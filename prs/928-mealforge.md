为家庭厨师、备餐爱好者和日常做饭人群打造 AI 智能周餐规划与采购清单助手，通过冰箱数字孪生、智能周餐规划、精准采购清单、口味学习引擎、烹饪引导五大核心功能，彻底解决"每天吃什么"的决策疲劳和食材浪费问题，让每顿饭都营养均衡、不重复、预算可控。

## 🎯 用户画像与场景分析

### 主要用户画像

**画像一：996上班族 "小忙"**
- 25-35 岁，互联网/金融行业，工作日极度忙碌
- 每天下班后最痛苦的事就是想吃什么
- 冰箱里经常有被遗忘的食材，周末集中清理浪费严重
- 月均外卖支出 ¥2000+，想自己做饭但缺乏规划能力
- 一人食或两人食，标准食谱份量总对不上

**画像二：全职妈妈 "饭团妈"**
- 30-40 岁，负责全家三口的日常饮食
- 每天重复做同样的菜，家人开始抱怨
- 想让孩子吃得营养均衡，但缺乏系统的营养知识
- 买菜凭经验，经常多买或少买，冰箱管理混乱
- 月初预算 ¥3000，但经常月底超支

**画像三：健身备餐党 "蛋白哥"**
- 22-30 岁，有规律的健身习惯
- 严格追踪宏量营养素（蛋白质/碳水/脂肪）
- 需要每周批量备餐，但食谱单调缺乏变化
- 想在"干净饮食"和"好吃"之间找到平衡
- 对食材成本敏感，需要性价比最高的蛋白质来源

**画像四：新手独居者 "泡面侠"**
- 18-25 岁，刚离开家独立生活
- 烹饪技能有限，只会煮面和煎蛋
- 不知道如何规划一周的买菜和做饭
- 想学做饭但缺乏系统引导
- 容易被短视频美食诱惑但实际操作失败率高

## 🔧 主要功能设计

### 功能一：冰箱数字孪生 🧊

**功能描述**：通过拍照识别、语音录入和手动输入三种方式，构建用户冰箱的数字模型，实时追踪库存状态和保质期。

**技术实现**：
```
录入方式：
1. 拍照识别
   - 单品拍照：对准食材拍摄，AI 识别品种、数量、新鲜度
   - 整体拍照：打开冰箱拍一张，AI 批量识别所有可见食材
   - 小票扫描：超市小票 OCR，自动提取购买的食材和数量
   - 条形码扫描：包装食品扫码入库

2. 语音录入
   - 自然语言输入："买了一斤西红柿、两盒鸡蛋、半斤五花肉"
   - AI 自动解析品种、数量、单位
   - 支持模糊描述："几个土豆"、"一些葱姜蒜"

3. 手动输入
   - 搜索式选择：输入关键词，从食材库中选择
   - 常用食材快捷按钮
   - 批量采购一键入库

追踪维度：
- 品种分类（蔬菜/肉类/蛋奶/水果/调料/主食/冷冻/饮品）
- 数量和单位（g/kg/个/盒/瓶/袋）
- 存放位置（冷藏/冷冻/常温/调料架）
- 购入日期和保质期
- 新鲜度评分（基于购入时间和品种特性）
- 剩余量估算（消耗后手动更新或 IoT 自动检测）
```

**AI 识别能力**：
```
识别范围：
- 新鲜蔬果：200+ 常见品种，准确率 95%+
- 肉类海鲜：100+ 品种，支持部位识别（鸡胸/鸡腿/鸡翅）
- 蛋奶制品：50+ 品种
- 调味料：80+ 常见调料
- 预包装食品：条形码 + OCR 双通道

新鲜度评估：
- 外观分析（颜色、形态、表面状态）
- 保质期倒计时（距离过期天数）
- 综合新鲜度评分（0-100）
- 推荐使用优先级
```

### 功能二：智能周餐规划 📋

**功能描述**：基于冰箱库存、口味偏好、营养目标和预算约束，自动生成个性化的一周餐食规划。

**规划引擎核心逻辑**：
```
输入参数：
- 冰箱当前库存（优先使用临期食材）
- 用户口味偏好画像
- 过敏/忌口信息
- 营养目标（热量/蛋白质/碳水/脂肪/微量元素）
- 预算范围（周预算或月预算）
- 用餐人数和场景（一人食/两人食/家庭餐/健身餐）
- 时间约束（工作日快手 vs 周末精致）
- 历史记录（避免重复近期吃过的菜）

约束求解：
- 每餐热量范围：早餐 300-500kcal，午餐 500-800kcal，晚餐 400-700kcal
- 每日蛋白质 ≥ 体重(kg)×1.0g（普通）/ ×1.5g（健身）
- 每日蔬菜 ≥ 300g（中国居民膳食指南推荐）
- 食材复用率 ≤ 30%（避免单调）
- 每周新增菜品 ≥ 3 道（保持新鲜感）
- 烹饪时间：工作日晚餐 ≤ 30min，周末 ≤ 60min

优化目标：
- 最小化额外采购成本（最大化利用现有库存）
- 最大化营养达标率
- 最大化口味匹配度
- 最小化食材浪费率
- 最大化菜品多样性
```

**规划交互流程**：
```
步骤1：一键生成
- AI 自动生成默认方案
- 展示一周餐食日历视图（早/午/晚/加餐）
- 标注每餐的营养数据和预计成本

步骤2：灵活调整
- 拖拽交换：拖拽某餐到另一天
- 锁定固定：锁定周一早餐为燕麦粥
- 指定替换："不想吃鱼，换成鸡肉"
- 主题日设置："周五外卖日"、"周末烘焙日"
- 批量调整："工作日都换成快手菜"

步骤3：营养总览
- 每日/每周营养柱状图
- 营养缺口提示（"本周维生素C摄入不足，建议增加西兰花或猕猴桃"）
- 营养均衡度评分（0-100）

步骤4：确认导出
- 确认方案 → 自动生成采购清单
- 保存为模板 → 下周可复用
- 分享给家人 → 家庭成员可查看和评论
```

**AI 食谱生成策略**：
```
生成方式优先级：
1. 精确匹配：食谱库中完全匹配库存+约束的食谱
2. 近似匹配：替换 1-2 种食材即可满足的食谱
3. AI 创作：LLM 基于库存和偏好创造新食谱
4. 经典推荐：营养师审核的高分经典食谱

食谱质量保障：
- 每道食谱经过营养学验证（基于 USDA + 中国食物成分表）
- 烹饪步骤经 AI 模拟验证（时间合理、步骤连贯）
- 用户评分反馈闭环（低于 3 分的食谱自动降权）
- 定期邀请营养师审核 AI 生成食谱
```

### 功能三：精准采购清单 🛒

**功能描述**：根据一周食谱自动生成精确采购清单，智能合并已有库存，按超市动线分组，支持多平台比价。

**清单生成逻辑**：
```
需求计算：
- 所需食材 = 食谱食材合集
- 需采购 = 所需食材 - 冰箱已有库存（考虑数量）
- 安全裕量：蔬菜类 +10%、肉类按实际需求量

分类分组：
├── 🥬 蔬菜区
│   ├── 番茄 × 3（500g）
│   ├── 西兰花 × 2（400g）
│   └── 菠菜 × 1（300g）
├── 🥩 肉类区
│   ├── 鸡胸肉 × 500g
│   └── 五花肉 × 300g
├── 🥚 蛋奶区
│   ├── 鸡蛋 × 1盒（10枚）
│   └── 牛奶 × 1L
├── 🍚 主食区
│   ├── 大米 × 5kg（整袋）
│   └── 全麦面包 × 1袋
├── 🧂 调料区
│   └── 生抽 × 1瓶（500ml）
└── ❄️ 冷冻区
    └── 速冻水饺 × 1袋
```

**智能比价系统**：
```
比价维度：
- 同一食材在不同平台的价格对比
- 不同规格的性价比计算（单价/g）
- 打折/促销信息整合
- 配送费和起送价考虑
- 到货时间预估

支持平台：
- 盒马鲜生
- 叮咚买菜
- 美团买菜
- 每日优鲜
- 京东到家
- 山姆会员店
- 拼多多（量大优惠）

比价输出示例：
┌──────────┬────────┬────────┬────────┬────────┐
│  食材     │ 盒马    │ 叮咚    │ 美团    │ 最优   │
├──────────┼────────┼────────┼────────┼────────┤
│ 鸡胸肉500g│ ¥12.9  │ ¥15.9  │ ¥11.9  │ 美团   │
│ 西兰花    │ ¥4.5/颗│ ¥3.9/颗│ ¥4.2/颗│ 叮咚   │

比单平台采购节省 ¥8.7（17%）
```

**历史价格追踪**：
- 记录每次采购价格，构建价格趋势曲线
- 标注食材的"价格低位"和"价格高位"时段
- 建议："鸡胸肉当前处于月度价格低位，可多囤一些冷冻"
- 季节性食材推荐："现在是番茄应季，价格比上月低 40%"

### 功能四：口味学习引擎 🧠

**学习维度**：
```

- 餐后评分（1-5 星）
- 标签反馈（太咸/太淡/太辣/太油腻/刚刚好）
- 收藏/屏蔽特定食谱
- 口味偏好问卷（初始设置）

隐式行为分析：
- 做了什么 vs 推荐了什么（跳过率）
- 修改了什么（减盐、加辣、换食材）
- 重复做的频率
- 每餐实际消耗比例（全吃完/剩一半）
- 采购行为（买了推荐清单外的食材 = 有隐藏需求）
- 浏览行为（在哪些食谱页面停留更久）
```

```
- "想吃更辣？加一勺郫县豆瓣酱，在炒番茄之前爆香"

计时器集成：
- 关键步骤自动启动计时器
- 倒计时提醒（"排骨还有5分钟收汁"）
- 多计时器并行运行
```
