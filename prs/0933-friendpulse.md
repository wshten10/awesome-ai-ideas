# FriendPulse - AI驱动的个性化社交关系维护与深度连接助手

> **Issue**: #933 | **状态**: 规格文档 | **版本**: v1.0

## 一、一句话定位

**FriendPulse** 是一款 AI 驱动的社交关系维护助手，通过关系健康度仪表盘、智能联系提醒、话题引擎与社交能量管理，帮助社交倦怠者和忙碌人群以最低心理负担维持有温度的人际关系。

## 三、核心功能模块

### 3.1 关系健康度仪表盘 📊

**核心理念**: 让关系状态可视化，像体检报告一样一目了然。

#### 数据维度

- **联系频率** (Contact Frequency): 基于历史互动记录计算
- **互动质量** (Interaction Quality): 基于对话长度、深度、情感分析
- **关系深度** (Relationship Depth): 基于共享经历、话题广度、情感投入
- **双向性** (Reciprocity): 双方主动联系的比例平衡度
- **近期趋势** (Trend): 过去30天的关系变化方向

#### 健康度评分模型

```
HealthScore = w1 × FrequencyScore + w2 × QualityScore 
            + w3 × DepthScore + w4 × ReciprocityScore + w5 × TrendScore

其中:
- FrequencyScore: 基于关系衰减模型计算（见3.6）
- QualityScore: LLM分析对话内容的情感丰富度（0-1）
- DepthScore: 关系图谱中的连接深度
- ReciprocityScore: |initiator_ratio - 0.5| 的倒数归一化
- TrendScore: 近30天vs前30天的变化率
- 权重 [w1-w5] 根据关系类型动态调整
```

#### 可视化呈现

- **总览页**: 所有关系的健康度分布（类似信用评分的色带）
- **关系卡片**: 每个重要关系的独立健康报告
- **趋势图**: 3个月/6个月/1年的关系变化曲线
- **预警标记**: 🔴 危险 🟡 注意 🟢 健康

### 3.2 智能联系提醒 ⏰

**核心理念**: 不是催你社交，而是在最佳时机轻轻推你一把。

#### 提醒触发条件

1. **时间衰减触发**: 根据关系衰减模型，当"社交温度"降至阈值时
2. **事件触发**: 对方生日、纪念日、重要考试/面试等
3. **话题触发**: 发现与对方相关的新话题（对方关注领域有新进展）
4. **情境触发**: 天气变化、节日、共同好友动态等
5. **周期触发**: 定期关系维护（如每月深度对话一次）

#### 反焦虑提醒设计

```
❌ 不好的提醒: "你已经15天没有联系张三了！"
✅ 好的提醒: "张三上周说在准备面试，现在差不多该出结果了，
           适合发条关心的消息~"
```

**设计原则**:
- 永远不使用负罪感驱动
- 提醒附带具体建议（不是空泛的"去联系吧"）
- 用户可设置"勿扰模式"和"社交预算"
- 提醒频率根据用户社交能量动态调整

### 3.3 话题引擎 💬

**核心理念**: 消除"不知道聊什么"的社交恐惧。

#### 话题生成流程

```
输入: 关系上下文 + 最近互动历史 + 对方兴趣标签 + 当前事件
  ↓
LLM 话题生成 (多轮推理)
  ↓
话题分类: [日常关心] [深度对话] [轻松闲聊] [共同回忆] [求助请教]
  ↓
话题筛选: 过滤敏感话题、根据关系深度匹配合适话题
  ↓
输出: 3-5个话题建议 + 推荐开场白 + 延伸话术
```

#### 话题示例

**关系**: 大学室友，关系健康度72%，最近30天未联系

| 类型 | 话题 | 推荐开场白 |
|------|------|-----------|
| 日常关心 | 最近工作怎么样 | "嘿，上周你说那个项目赶得挺急的，搞定了吗？" |
| 共同回忆 | 大学时的某段经历 | "突然想起咱们大四那个通宵赶论文的晚上😂" |
| 话题分享 | 对方关注的领域 | "看到个关于AI绘画的帖子，想起你之前在玩这个" |

#### 个性化适配

- **关系深度**: 浅层关系 → 轻松话题；深层关系 → 可涉及情感话题
- **对方状态**: 对方最近压力大 → 避免沉重话题
- **时间节点**: 深夜 → 不建议开启需要长时间回复的话题
- **平台差异**: 微信风格 vs 朋友圈评论风格

### 3.4 社交能量管理 🔋

**核心理念**: 社交是一种有限资源，需要像管理精力一样管理社交能量。

#### 社交能量模型

```
SocialEnergy(t) = BaseEnergy - Σ(SocialCost_i) + Σ(SocialRecharge_i)

SocialCost_i: 每次社交互动消耗的能量
  - 取决于: 关系深度、互动质量、是否被迫、当前情绪状态
  
SocialRecharge_i: 社交后的能量恢复
  - 取决于: 互动是否愉快、是否获得情感支持、是否有深度连接时刻
```

#### 能量仪表盘

- **当前能量值**: 0-100% 的实时显示
- **能量预测**: 基于日程和待办社交事项的预测
- **最优社交时段**: 根据历史数据推荐最佳社交时间
- **低能量模式**: 当能量低于阈值时，自动简化提醒（只保留最重要的）

#### 社交预算

```
用户设置: 本周社交预算 = 中等
系统建议: 
  - 今日可用社交额度: 3次轻度互动 或 1次深度对话
  - 已使用: 1次（早上回复了同事消息）
  - 剩余: 2次轻度互动
  - 建议: 今晚给妈妈打个电话（关系优先级高 + 周末充电效果好）
```

### 3.5 重要时刻守护 🛡️

**核心理念**: 永远不错过真正重要的时刻。

#### 时刻识别

| 类型 | 来源 | 提前时间 | 动作 |
|------|------|---------|------|
| 生日 | 手动录入/日历同步 | 7天/3天/当天 | 提醒 + 礼物建议 |
| 纪念日 | 手动录入 | 3天/1天/当天 | 提醒 + 纪念方式建议 |
| 重要事件 | 对话中提取/手动录入 | 根据事件类型 | 关心 + 跟进提醒 |
| 人生转折 | 对话分析 | 持续关注 | 深度关怀提醒 |

#### 智能提取

通过 LLM 分析对话历史，自动识别并记录重要信息：

```
对话片段: "下周五我有个面试，有点紧张..."
提取: {event: "面试", date: "下周五", emotion: "紧张"}
后续动作: 面试当天提醒关心，面试后2天跟进结果
```

### 3.6 关系衰减模型 📉

**核心理念**: 用数学模型量化关系随时间的变化。

#### 指数衰减模型

```
RelationshipStrength(t) = S₀ × e^(-λt) + RecentBoost(t)

其中:
- S₀: 关系的基础强度（由历史互动累计）
- λ: 衰减系数（因关系类型而异）
  - 亲密家人: λ ≈ 0.001（极慢衰减）
  - 好朋友: λ ≈ 0.01
  - 普通朋友: λ ≈ 0.03
  - 同事: λ ≈ 0.05
- t: 距上次深度互动的天数
- RecentBoost(t): 最近轻互动（点赞、评论）的加成
```

#### 恢复机制

```
RecoveryModel:
- 一次深度对话（>30分钟）: 恢复至 S₀ × 0.85
- 一次中等互动（10-30分钟）: 恢复至 S₀ × 0.60
- 一次轻互动（<10分钟）: 恢复至 S₀ × 0.30
- 持续3次轻互动: 累计恢复至 S₀ × 0.70
- 连续7天每日轻互动: 恢复至 S₀ × 0.95
```

### 4.4 隐私架构

```
隐私设计原则:
├── 端到端加密: 所有对话数据端到端加密存储
├── 本地优先: 核心计算在设备端完成，最小化云端数据
├── 数据最小化: 只收集必要数据，不收集对话内容原文
├── 用户主权: 完整的数据导出/删除功能
├── 透明可控: 所有AI分析结果可解释、可纠正
└── 匿名化: AI处理时使用匿名化数据，可配置处理模式
```

**隐私级别配置**:

| 级别 | 本地处理 | 云端处理 | 功能完整度 |
|------|---------|---------|-----------|
| 🔒 最高隐私 | 100% | 0% | 基础功能 |
| 🔐 高隐私 | 80% | 20%（仅LLM调用） | 大部分功能 |
| 🔓 标准 | 60% | 40% | 全部功能 |
| ⚡ 性能优先 | 40% | 60% | 全部功能+增强 |

### 5.3 反焦虑设计细节

- **无数字焦虑**: 不显示具体的天数倒计时（如"已经23天没联系了"）
- **正面框架**: 用"上次联系是X月X日"代替"已经X天没联系"
- **柔软语言**: 所有提醒使用温暖、非指令性语言
- **低压力操作**: 支持快捷回复模板、一键发送关心表情
- **社交假期模式**: 一键开启，暂停所有提醒3-14天

## 十一、设计哲学

### 反焦虑宣言

> FriendPulse 不相信"社交KPI"。我们不认为你需要每天联系10个人才能维持关系。
> 我们相信：**一段好的关系，不在于联系的频率，而在于每次联系的温度。**
>
> 我们不会用负罪感驱动你社交。我们会在你精力充沛时建议你联系最需要关心的那个人。
> 我们会在你疲惫时安静退场，不增加任何心理负担。
>
> **FriendPulse 的终极目标，是让你不需要 FriendPulse。**

### 核心原则

1. **温度 > 频率**: 一次有深度的对话胜过十次寒暄
2. **主动 > 被动**: 帮助用户成为社交的主导者，而非被动响应者
3. **质量 > 数量**: 维护少数重要关系，而非追求社交网络规模
4. **隐私 > 功能**: 宁可少一个功能，也不多收集一分数据
5. **减负 > 加载**: 每个功能都应该减少用户的心理负担，而不是增加
