# 🌱 GreenThumb — AI 微型花园规划与植物养护教练 (#909)

## 概述

为城市公寓居民、阳台园艺爱好者和绿植新手打造 AI 智能微型花园规划与植物养护教练平台，通过环境分析引擎、拍照诊断、智能种植规划、养护日历、阳台 3D 规划器五大核心功能，彻底解决"植物杀手"循环和阳台空间浪费问题，让每个人都能轻松拥有属于自己的绿色治愈空间。

## 📋 相关信息

- **相关 Issue**: #909
- **目标用户**: 城市公寓居民、阳台园艺爱好者、绿植新手
- **核心价值**: 从养啥死啥和阳台空间浪费到 AI 驱动的个性化微型花园规划与植物养护教练
- **技术评估**: 实现难度中等偏上，多模态识别和 3D 渲染为核心技术挑战
- **市场潜力**: 全球室内植物市场 2028 年预计达 $25B+，中国城市阳台园艺渗透率快速上升

## 🔧 主要功能设计

### 功能一：环境分析引擎 🌤️

**功能描述**：基于用户输入的阳台参数和地理位置，智能评估环境适合度，推荐最佳植物组合。

**技术实现**：
```
输入参数：
- 阳台朝向（东/南/西/北/东南/西南）
- 面积（平方米）
- 楼层高度
- 遮挡情况（有无对面建筑遮挡）
- 是否封闭/开放
- 地理位置（GPS 或城市选择）

分析维度：
- 日照时长估算（基于朝向+遮挡+季节）
- 温度范围（基于楼层+地理位置+季节）
- 通风条件（基于朝向+楼层+遮挡）
- 湿度评估（基于地理位置+季节）
- 综合环境评分（0-100）

输出结果：
- 环境等级（优秀/良好/一般/较差）
- 推荐植物列表（按适合度排序）
- 不推荐植物列表（附原因说明）
- 环境改善建议
```

**AI 算法逻辑**：
- 基于植物数据库中 2000+ 品种的光照、温度、湿度需求矩阵
- 结合地理位置的气候数据（年均温、极端温度、降水模式）
- 使用协同过滤算法，参考同城市同朝向用户的成功案例
- 季节性动态调整，夏季和冬季推荐不同品种

**数据来源**：
- 中国气象局开放数据 API
- OpenWeatherMap 商业 API
- 用户贡献数据（UGC 环境反馈）

### 功能二：拍照诊断系统 🔬

**功能描述**：多模态 AI 识别植物品种和健康状态，提供精准诊断和治疗方案。

**诊断流程**：
```
步骤1：图像采集
- 支持叶片特写、整株照片、根部照片
- 自动质量检测（模糊、过曝、过暗提示重拍）
- 多角度拍摄建议引导

步骤2：品种识别
- 多模态 LLM（GPT-4o / Claude Vision）识别品种
- 置信度评分，低于 80% 提供候选列表
- 识别结果存入用户植物档案

步骤3：健康诊断
- 视觉特征分析（叶片颜色、斑点、卷曲、萎蔫）
- 病虫害模式匹配（200+ 常见病虫害数据库）
- 养护问题排查（浇水、施肥、光照、温度）

步骤4：治疗方案
- 紧急处理措施（24小时内执行）
- 短期治疗方案（1-2周）
- 长期养护调整建议
- 所需物品清单（肥料、药剂、工具）
```

**常见问题诊断能力**：
| 问题类型 | 可识别症状 | 准确率目标 |
|---------|-----------|-----------|
| 浇水问题 | 黄叶、萎蔫、根系腐烂 | 92% |
| 病虫害 | 白粉病、红蜘蛛、蚜虫、黑斑病 | 88% |
| 缺素症状 | 缺氮、缺铁、缺镁、缺磷 | 85% |
| 光照问题 | 徒长、叶缘灼伤、褪色 | 90% |
| 温度问题 | 冻伤、热害 | 87% |

### 功能三：智能种植规划器 📐

**功能描述**：根据环境条件和个人偏好，生成个性化的植物搭配方案和购买清单。

**规划维度**：
- **功能偏好**：观花 / 观叶 / 可食用 / 香草 / 净化空气
- **养护难度**：入门级 / 进阶级 / 专家级
- **预算范围**：经济型 / 标准型 / 高端型
- **空间利用**：垂直种植 / 悬挂式 / 地面摆放 / 组合方案
- **色彩搭配**：季节性色彩方案推荐
- **植物相性**：避免相克组合（如薄荷与其他香草）

**输出内容**：
```
种植方案包含：
1. 推荐植物列表（图片、名称、适合度评分、预估价格）
2. 购买清单（按优先级排序，附购买渠道推荐）
3. 种植时间表（最佳种植月份、生长周期）
4. 搭配效果图（2D 俯视图 / 3D 渲染图）
5. 预期效果描述（何时开花、何时结果）
6. 投资回报估算（可食用植物的产量预估）
```

### 功能四：养护日历与提醒系统 📅

**功能描述**：基于每株植物的生长阶段、当地天气和用户习惯，推送个性化养护任务。

**任务类型**：
- **日常任务**：浇水（基于天气预报智能调整频率）、通风
- **周期任务**：施肥（生长季/休眠季不同频率）、修剪、清洁叶片
- **季节任务**：换盆（春季）、入室/出室（秋冬季）、防冻措施
- **应急任务**：极端天气预警（高温、霜冻、暴雨）、病虫害预防

**智能特性**：
- 结合 3 天天气预报动态调整浇水计划
- 学习用户执行习惯，优化提醒时间
- 支持多植物批量操作（"今天所有植物都需要浇水"）
- 假期模式：自动生成出行期间的养护方案（自动浇水器设置建议、拜托邻居的注意事项）

**数据模型**：
```json
{
  "plant_id": "user_001_monstera_01",
  "care_schedule": {
    "watering": {
      "base_interval_days": 7,
      "weather_adjusted": true,
      "last_done": "2026-04-08",
      "next_due": "2026-04-14",
      "soil_moisture_threshold": 0.3
    },
    "fertilizing": {
      "frequency": "biweekly",
      "season": "growing",
      "type": "balanced_npk",
      "last_done": "2026-04-01"
    }
  }
}
```

### 功能五：阳台 3D 规划器 🏗️

**功能描述**：拖拽式阳台布局工具，AI 优化植物摆放位置以最大化光照利用。

**核心功能**：
- **阳台建模**：输入尺寸自动生成 3D 阳台模型
- **植物库**：300+ 常见阳台植物的 3D 模型（含不同生长阶段）
- **拖拽摆放**：直觉式拖拽植物到阳台任意位置
- **光照模拟**：基于朝向和时间，实时模拟光照分布
- **AI 优化**：一键优化按钮，AI 自动调整位置使每株植物获得最佳光照
- **生长模拟**：时间轴滑块，预览 1/3/6/12 个月后的生长效果
- **截图分享**：生成精美效果图分享到社交平台

**技术实现**：
- 前端渲染：Three.js + React Three Fiber
- 光照计算：基于太阳位置算法（SPA）+ 遮挡计算
- 植物生长模型：基于 L-System 的程序化生长模拟
- 3D 模型：GLTF 格式，支持 LOD（Level of Detail）优化

### 数据库设计

**核心数据表**：

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY,
    phone VARCHAR(20) UNIQUE,
    nickname VARCHAR(50),
    location GEOGRAPHY(POINT, 4326),
    balcony_config JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 植物档案表
CREATE TABLE plant_profiles (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    species_id UUID REFERENCES plant_species(id),
    nickname VARCHAR(50),
    location_in_balcony JSONB,
    planted_date DATE,
    health_status VARCHAR(20) DEFAULT 'healthy',
    growth_stage VARCHAR(20),
    custom_care_notes TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 养护记录表
CREATE TABLE care_logs (
    id UUID PRIMARY KEY,
    plant_id UUID REFERENCES plant_profiles(id),
    action_type VARCHAR(20), -- watering, fertilizing, pruning, repotting
    notes TEXT,
    photo_url TEXT,
    performed_at TIMESTAMP DEFAULT NOW()
);

-- 诊断记录表
CREATE TABLE diagnoses (
    id UUID PRIMARY KEY,
    plant_id UUID REFERENCES plant_profiles(id),
    image_urls TEXT[],
    species_identified UUID REFERENCES plant_species(id),
    confidence_score FLOAT,
    issues JSONB,
    treatment_plan JSONB,
    diagnosed_at TIMESTAMP DEFAULT NOW()
);

-- 植物物种知识库
CREATE TABLE plant_species (
    id UUID PRIMARY KEY,
    common_name VARCHAR(100),
    scientific_name VARCHAR(100),
    family VARCHAR(100),
    category VARCHAR(50),
    care_requirements JSONB,
    light_needs VARCHAR(20),
    water_needs VARCHAR(20),
    temperature_range JSONB,
    humidity_range JSONB,
    growth_rate VARCHAR(20),
    max_height_cm INT,
    toxic_to_pets BOOLEAN DEFAULT FALSE,
    companion_plants UUID[],
    incompatible_plants UUID[],
    common_issues JSONB,
    images TEXT[],
    created_at TIMESTAMP DEFAULT NOW()
);
```
