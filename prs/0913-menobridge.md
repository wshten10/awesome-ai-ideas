# MenoBridge - AI 更年期健康管理伙伴

> **一句话卖点**：专为 40-58 岁围绝经期/更年期女性设计，语音记录症状 → AI 识别症状模式 → 个性化循证建议 + 职场应对策略，从「默默忍受」到「科学管理」

## 概述

基于 Issue #913 创建。更年期平均持续 4-10 年，全球约 15 亿女性正在经历，但专门服务这一群体的数字健康产品极度稀缺。现有产品要么偏重处方药（Evernow/Winona），要么是企业级福利（Maven Clinic），面向中文用户的 C 端产品几乎空白。

本产品定位「**全周期更年期健康管理伙伴**」，核心路径：**语音/文字记录症状 → AI 结构化分析 → 模式识别 → 个性化建议 + 医疗/职场/生活场景应对**。

**产品经理评分**：4.0/5（蓝海市场，需求刚性，冷启动依赖医学 KOL 信任背书）
**增长黑客评分**：8.5/10（话题本身自带传播势能，「终于有人关注」的共鸣驱动自发分享）
**技术专家评分**：6/10（架构清晰，主要风险在医疗合规和 NLP 症状识别精度）

**核心价值**：不是又一个健康 App，而是更年期女性 4-10 年旅程中的全天候 AI 伙伴——识别你看不到的症状关联，在你没意识到时预警恶化趋势，在职场尴尬时提供即时应对策略。

## 竞品分析

### 主要竞争对手

| 产品 | 定位 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|-------------|
| **Maven Clinic** | 企业级女性健康 | 覆盖全生命周期，企业付费 | 不面向 C 端个人用户，中文市场无 | C 端直接可用，中文原生 |
| **Evernow/Winona** | 更年期处方药 | 医疗专业，有处方权 | 仅解决激素问题，不含症状管理/职场支持 | 全方位管理，非药物导向 |
| **Flo/Clue** | 月经/生育追踪 | 用户量大，数据积累深 | 更年期功能薄弱，非核心场景 | 专为更年期设计，深度场景覆盖 |
| **薄荷健康** | 中国健康管理 | 本土化好，用户基础大 | 无更年期专项功能 | 更年期垂直，AI 症状模式识别 |
| **小红书/抖音科普** | UGC 内容 | 内容丰富，免费 | 信息碎片化，伪科学泛滥，无个性化 | 循证医学 + 个性化追踪 |

### 护城河分析
1. **数据壁垒**：2-4 周症状日志后，AI 洞察质量显著提升，用户迁移成本高
2. **中文市场先发**：面向 2 亿中国更年期女性的首个 AI 专项产品
3. **医学信任链**：与妇科 KOL/医院合作建立的专业信任难以复制
4. **社区网络效应**：更年期话题的私密性决定了信任社区的不可替代性

## 功能设计

### 核心功能

**1. 多模态症状追踪引擎**

- **语音记录**：「今天又潮热了三次，下午心慌，晚上翻来覆去睡不着」→ AI 自动结构化
- **文字记录**：支持自由文本，LLM 提取症状类型、严重程度、持续时间、触发因素
- **结构化表单**：MVP 阶段主推，降低 NLP 精度压力，支持自由文本补充
- **30+ 症状覆盖**：潮热、盗汗、失眠、情绪波动、关节痛、心悸、头痛、皮肤干燥、泌尿问题等
- **触发因素标记**：饮食、压力、运动、环境温度等关联标记

**症状结构化设计**：
```
输入：「最近总出汗，睡不好，脾气也变差了」
输出：
{
  "date": "2026-04-10",
  "symptoms": [
    {"type": "潮热/盗汗", "severity": "moderate", "time": "日间"},
    {"type": "失眠", "severity": "moderate", "context": "入睡困难"},
    {"type": "情绪波动", "severity": "mild", "context": "易怒"}
  ],
  "triggers": ["未标记"],
  "notes": "用户描述较模糊，建议补充具体时间和频率"
}
```

**2. 个性化洞察生成（2-4 周数据后）**

- **趋势分析报告**：周报/月报，可视化症状频率、严重程度变化趋势
- **模式识别**：发现用户未注意到的症状关联（如「每次吃辣后潮热加重」「运动后睡眠质量提升 40%」）
- **循证建议**：基于医学文献 RAG，提供有证据等级标注的建议（A/B/C 级证据）
- **预警机制**：症状恶化趋势预警，建议及时就医并准备就诊清单

**3. 智能医疗匹配**

- 根据症状组合推荐合适的专科（妇科内分泌/心理科/骨科等）
- 生成就诊准备清单：症状摘要、趋势图表、用药记录、过敏史
- 支持导出 PDF 供医生参考
- 合作医生资源库（分城市）

**4. 职场场景助手**

- **即时应对策略**：会议中潮热怎么办？情绪波动时如何快速平复？
- **匿名沟通模板**：帮助向 HR/主管申请合理调整（远程办公、弹性工时、温度控制）
- **权益知识库**：各国/地区更年期职场保护法规
- **情境模拟**：AI 角色扮演，练习与主管沟通更年期需求

### 功能优先级

| 阶段 | 功能 | 预计时间 |
|------|------|----------|
| MVP | 症状追踪（结构化表单+自由文本）+ 周报摘要 + 知识库问答 | 4 周 |
| V1 | 医疗匹配 + 职场助手 + 社区 | +6 周 |
| V2 | 可穿戴设备集成 + 营养运动联动 + 家庭教育模块 | +8 周 |

### 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **LLM** | GPT-4o + 本地 Llama 3 8B | 云端复杂推理 + 端侧隐私敏感任务 |
| **RAG** | LangChain + pgvector | 医学文献检索，证据等级标注 |
| **前端** | React Native (Expo) | 跨平台，40-58 岁用户需要简单易用 |
| **后端** | FastAPI + PostgreSQL | 类型安全，异步支持好 |
| **认证** | Firebase Auth | 快速集成，支持手机号登录 |
| **存储** | 本地优先 + 可选云同步 | 隐私保护，敏感数据加密 |

### NLP 症状识别策略

**MVP 阶段（务实方案）**：
- 主推结构化表单，用户选择症状类型 + 滑块评估严重程度
- 自由文本作为补充，LLM 提取后让用户确认
- 避免冷启动阶段因 NLP 精度不足影响用户体验

**V1 阶段（语音优先）**：
- Whisper 语音转文字 + LLM 结构化
- 2-4 周用户数据 fine-tune 症状识别模型
- 用户确认机制持续提升精度

### 医疗合规策略

| 阶段 | 合规范围 | 具体措施 |
|------|----------|----------|
| MVP | 基础数据保护 | 端到端加密 + 用户授权 + 数据不出境 |
| V1 | 中国个人信息保护法 | 隐私影响评估 + 数据本地化存储 |
| V2 | HIPAA/GDPR | 国际标准合规，支持海外用户 |

**关键原则**：
- 不提供诊断，只提供信息和建议
- 所有建议标注证据等级和来源
- 明确提示「AI 建议 ≠ 医疗诊断，请咨询医生」
- 数据最小化收集原则

## MVP 验证方案

### 核心验证链路（4 周内）

```
用户语音/文字描述症状
    → LLM 结构化提取
    → 存储症状日志
    → 2周后生成趋势摘要
    → RAG 检索相关医学知识
    → 生成个性化建议
```

### 技术验证指标

- 症状结构化准确率 > 85%（用户确认率）
- 周报生成延迟 < 5 秒
- 医学建议相关性评分 > 4/5（医生评审）

### 用户验证指标

- 周留存 > 30%（4 周内）
- 日志记录频率 > 3 次/周
- NPS > 40（初期目标）
