## 1. 项目概述与痛点量化

### 1.1 项目定位

NestHunter 是一个面向中国城市租房者的 AI 驱动全链路租房决策平台，覆盖从房源发现、真实性核验、合同审查到搬家执行的完整生命周期。平台通过多平台数据聚合、机器学习模型与大语言模型的深度协同，解决租房市场中信息不对称、信任缺失、决策成本高等核心问题。

### 1.2 目标用户画像

| 用户群体 | 核心特征 | 核心需求 |
|---------|---------|---------|
| 应届毕业生 | 首次租房，预算紧张，缺乏经验 | 低成本、可信房源、合同指导 |
| 城市白领 | 通勤敏感，对居住品质有要求 | 通勤优化、周边配套匹配、性价比 |
| 频繁搬家者 | 年搬家1-2次，流程熟悉但耗时 | 快速筛选、搬家流程自动化 |
| 异地求职者 | 对目标城市不熟悉，远程看房 | 虚拟看房辅助、区域评估 |

### 1.3 市场规模

- 中国租房市场规模超 **2.2万亿元**（2024年），年均增长约8%
- 全国租房人口超 **2.3亿**，其中新市民和青年人占比超70%
- 一线城市年均换租率约 **35%**，即每3年至少换一次房
- 在线租房平台渗透率仅约 **40%**，大量交易仍依赖线下中介

### 1.4 痛点量化分析

#### 痛点一：房源信息极度分散

- 用户平均需要访问 **3-5个平台** 才能覆盖基本房源供给
- 同一房源在不同平台的价格差异可达 **15%-30%**
- 跨平台比价、交叉验证平均耗时 **8-12小时/次**

#### 痛点二：虚假与过期房源泛滥

- 主要平台虚假房源比例估计在 **15%-40%**（低价引流型）
- 过期未下架房源占比约 **20%-25%**
- 用户平均被骗/无效看房 **3-5次** 后才找到真实房源

#### 痛点三：通勤与配套评估依赖手动

- 通勤时间计算需结合地图API + 地铁线路 + 步行距离，手动评估误差大
- 周边配套（医院、超市、学校、健身房等）缺乏结构化评分体系
- 约 **68%** 的租客表示入住后发现通勤或配套不如预期

#### 痛点四：合同条款复杂，隐形费用多

- 租房合同平均长度 **15-30页**，法律术语密集
- 常见隐形费用：物业费分摊、维修责任模糊、提前解约违约金、涨租条款
- 约 **45%** 的租客遭遇过合同相关纠纷

#### 痛点五：搬家流程琐碎易遗漏

- 搬家涉及 **20-35项** 独立任务（退租检查、宽带迁移、地址变更、保洁等）
- 约 **60%** 的租客至少遗漏1项重要事项
- 平均搬家准备周期 **7-14天**，任务协调效率低

#### 3.2.2 LightGBM分类器详细设计

**特征工程（120维）：**

| 特征组 | 特征数 | 示例特征 |
|--------|--------|---------|
| 价格特征 | 15 | 绝对价格、小区均价比、面积单价、同户型中位数偏差 |
| 地理特征 | 20 | 经纬度、商圈编码、地铁距离、POI密度、学区评分 |
| 文本特征 | 40 | 标题长度、描述长度、关键词TF-IDF top-50、BERT 768维→PCA 20维 |
| 图像特征 | 25 | 图片数量、平均亮度/饱和度、CLIP嵌入PCA 15维、pHash匹配数 |
| 发布者特征 | 15 | 账号年龄、历史发布数、历史被举报率、回复速度 |
| 时间特征 | 5 | 发布时段、距上次修改时间、更新频率 |

**训练策略：**

- 正样本：已验证成交房源（链家成交数据 + 用户反馈确认）
- 负样本：用户举报虚假房源 + 价格异常房源 + 平台下架房源
- 采样策略：SMOTE过采样少数类 + Tomek Links欠采样
- 评估指标：Precision@95% Recall ≥ 90%，F1 ≥ 0.88

#### 3.2.3 图片去重系统

```python
class ImageDedup:
    """
    多hash算法融合的图片去重系统
    用于检测不同房源复用同一组图片（虚假房源常见手段）
    """
    HASH_ALGORITHMS = {
        "phash": PerceptualHash(hash_size=16),  # 感知hash，抗缩放
        "dhash": DifferenceHash(hash_size=16),   # 差值hash，抗亮度
        "ahash": AverageHash(hash_size=16),      # 均值hash，最快
    }
    
    def compute_fingerprint(self, image_path: str) -> np.ndarray:
        """计算融合指纹：三种hash拼接 + CLIP嵌入PCA"""
        hashes = [algo(image_path) for algo in self.HASH_ALGORITHMS.values()]
        clip_embedding = self.clip_model.extract(image_path)  # 768维
        clip_reduced = PCA(n_components=32).transform(clip_embedding)
        return np.concatenate([*hashes, clip_reduced])  # 80维指纹
    
    def find_similar(self, fingerprint: np.ndarray, 
                     threshold: float = 0.92) -> List[MatchResult]:
        """在Milvus中检索相似图片"""
        return self.milvus_client.search(
            collection="listing_images",
            data=[fingerprint],
            top_k=10,
            metric_type="IP",
            params={"nprobe": 32},
            threshold=threshold,
        )
```

#### 3.3.3 技术实现要点

- **Prompt Engineering**：合同分析采用CoT（Chain-of-Thought）推理，先结构化提取再逐条风险判定
- **幻觉控制**：风险条款判定必须引用合同原文片段，禁止模型自行编造条款内容
- **法律知识增强**：通过RAG检索最新地方法规作为上下文，提升合规判断准确性
- **性能优化**：合同文本预分chunk（按条款段落），并行推理后聚合，平均处理时间 < 30秒/份

```python
                "驾照/车辆登记",
                "紧急联系人地址",
            ],
            "dependencies": ["move_in"],
            "deadline_offset": 14,  # 入住后14天内完成
        },
        "cleaning": {
            "name": "保洁",
            "dependencies": ["keys_received"],
            "options": ["self_clean", "professional_clean"],
            "estimated_cost": {"self": 0, "professional": "200-500元"},
        },
    }
    
    def generate_dag(self, user_profile: UserProfile, 
                     move_in_date: datetime) -> TaskDAG:
        """
        根据用户画像动态生成个性化任务DAG
        - 用户有车 → 增加停车位确认、车辆登记变更
        - 用户有宠物 → 增加宠物登记、宠物友好设施确认
        - 用户跨城市 → 增加社保转移、医保备案
        """
        tasks = self._select_tasks(user_profile)
        dag = self._build_dependency_graph(tasks, move_in_date)
        return self._optimize_schedule(dag, user_profile.availability)
```

## 4. 竞品分析

### 4.1 竞品全景对比

| 维度 | 链家/贝壳 | 自如 | 豆瓣租房 | 58同城 | **NestHunter** |
|------|----------|------|---------|--------|---------------|
| 房源来源 | 自有+合作 | 自营 | UGC | 平台聚合 | **多平台聚合** |
| 真实性 | 较高（VR看房） | 高（自营） | 低（无审核） | 低 | **AI多层核查** |
| 合同服务 | 标准合同 | 标准合同 | 无 | 无 | **AI风险扫描** |
| 通勤计算 | 简单直线距离 | 无 | 无 | 无 | **多方式实时计算** |
| 搬家服务 | 无 | 有（增值服务） | 无 | 无 | **全自动任务编排** |
| AI能力 | 基础推荐 | 基础匹配 | 无 | 无 | **全链路AI驱动** |
| 目标用户 | 全年龄段 | 年轻白领 | 文艺青年 | 全年龄段 | **租房新人群** |
| 收费模式 | 佣金制 | 服务费制 | 免费 | 付费推广 | **免费+SaaS** |

### 4.2 链家/贝壳找房

**优势：**
- 真房源体系行业标杆，VR看房技术领先
- 数据积累深厚，小区成交历史数据丰富
- 线下门店网络覆盖广泛，经纪人服务体系成熟

**劣势：**
- 佣金成本高（一般一个月租金）
- 封闭生态，房源仅限自有平台
- 合同服务标准化程度高但灵活性不足
- 无AI驱动的个性化推荐能力

**NestHunter差异化：** 聚合链家数据但不依赖其生态，AI真实性核查可作为跨平台可信度基准

### 4.3 自如

**优势：**
- 自营房源品质统一，装修标准化
- 增值服务完整（保洁、维修、搬家）
- 年轻化品牌定位，用户口碑较好

**劣势：**
- 房源溢价明显（普遍高于同小区15%-25%）
- 服务费+管理费构成额外成本
- 合同条款偏利于平台方
- 长租公寓爆雷事件影响用户信任

**NestHunter差异化：** 帮用户对比自如vs非自如房源的TCO（总拥有成本），合同扫描器可识别不利于租客的自如条款

### 4.4 豆瓣租房

**优势：**
- 个人直租房源丰富，无中介费
- 社区氛围浓厚，用户自发审核
- 城市租房小组活跃度高

**劣势：**
- 无任何审核机制，虚假信息泛滥
- 信息结构化程度极低（纯文本帖子）
- 无配套服务（合同、搬家等）
- 用户体验原始，搜索能力弱

**NestHunter差异化：** 抓取豆瓣租房数据后进行AI结构化和真实性核查，将UGC噪音转化为可用信号

### 4.5 58同城/安居客

**优势：**
- 房源量最大，覆盖城市最广
- 分类信息生态完整
- 移动端流量优势明显

**劣势：**
- 虚假房源问题最严重（"刺客房源"现象严重）
- 付费推广机制导致劣币驱逐良币
- 无AI能力，推荐算法简单
- 用户体验差，广告干扰多

**NestHunter差异化：** 对58房源进行最严格的AI真实性核查，将高可信房源优先推荐，构建平台内的可信房源子集
