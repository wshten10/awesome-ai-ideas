# AI 简历诊断师

> **一句话卖点**：海投 300 份简历 0 回复的求职者，拍照上传简历 3 秒收到精准诊断，改 3 版拿到 5 个面试

## 概述

基于 Issue #140 创建。海投简历石沉大海是中国 8000 万+年度求职者最深层的焦虑。用户的核心痛点不是"不知道怎么写简历"，而是"不知道自己的简历哪里有问题"——网上的模板千篇一律，朋友的反馈不专业，付费简历服务 299-999 元且反馈慢。

本产品通过「秒级 AI 诊断 + JD 对标分析 + 改前改后预览」三位一体解决这一痛点。AI 3 秒给出问题清单（缺少数据支撑、关键词不足、ATS 友好度低等），对标目标岗位 JD 指出技能匹配差距，并模拟 HR 视角展示修改前后的印象分变化。

**核心价值**：不只是告诉你"项目经历要写具体"（太抽象），而是告诉你"第 3 段缺少数据，建议：用户数从 X 增长到 Y，转化率提升 Z%"（具体可执行）。

## 竞品分析

### 主要竞争对手

| 产品 | 定位 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|-------------|
| **超级简历** | 简历排版工具 | 模板丰富，WYSIWYG 编辑 | 本质是排版，无诊断能力 | 诊断 + 改进建议，不只是排版 |
| **简历大师** | 简历优化服务 | 一对一人工服务 | ¥299-999 元，周期 3-5 天 | 3 秒出结果，¥9.9 元/次 |
| **JobScan** | ATS 关键词匹配 | 英文 ATS 优化成熟 | 中文支持差，无具体修改建议 | 中国本土化 + 具体可执行建议 |
| **Kickresume** | AI 简历生成 | AI 生成简历文本 | 需要用户从零开始填写 | 基于已有简历诊断，降低门槛 |
| **ChatGPT 通用对话** | 通用 AI 助手 | 免费，能力全面 | 需要用户自己写 prompt，无结构化输出 | 一键诊断，结构化报告，无需 prompt 技巧 |

### 我们的护城河
1. **效果追踪闭环**：用户修改后投递 → 记录回复率 → 优化诊断算法（竞品均缺失）
2. **JD 对标数据库**：积累各行业岗位 JD → 技能关键词匹配度越来越准
3. **改前改后对比**：直观展示修改效果，天然适合社交传播（小红书/抖音）
4. **中国本土化**：互联网/金融/外企/国企不同风格的简历规范，中文简历特有问题（照片、年龄、政治面貌等）

## 功能设计

### 核心功能
1. **秒级诊断** - 上传简历（PDF/Word/截图），AI 3 秒给出 5 大维度问题清单
2. **JD 对标分析** - 粘贴目标岗位 JD，计算匹配度 + 差距清单
3. **改前改后预览** - 高亮修改部分（红色删除/绿色新增），模拟 HR 视角打分
4. **行业定制引擎** - 互联网（STAR 法则）、金融（数据导向）、外企（英文简历规范）、国企（规范格式）
5. **匿名化处理** - 自动脱敏姓名/手机号/邮箱，保护隐私
6. **ATS 友好度检测** - 检测简历是否会被招聘系统过滤（格式/关键词/结构）

### 诊断维度（5 大维度 × 20+ 子项）

| 维度 | 子项示例 | 权重 |
|------|---------|------|
| **内容完整度** | 教育背景、工作经历、项目经验、技能清单是否齐全 | 20% |
| **数据支撑度** | 是否有量化数据（用户数/增长/转化率/营收） | 25% |
| **关键词匹配** | 与目标 JD 的技能关键词重合度 | 20% |
| **表达专业度** | 动词使用（主导 vs 参与）、STAR 法则应用 | 20% |
| **ATS 友好度** | 格式兼容性、关键词密度、结构清晰度 | 15% |

### 差异化优势
| 现有方案 | 本产品 |
|---------|--------|
| 网上简历模板，不知道哪个适合自己 | 上传简历，AI 诊断你的简历具体问题 |
| "项目经历要写具体"（太抽象） | "第3段缺少数据，建议：用户数从X增长到Y，转化率提升Z%"（具体可执行） |
| 找朋友帮忙看，反馈慢且不专业 | 24/7 秒级响应，基于真实 HR 筛选逻辑 |
| 付费简历服务 ¥299-999 元 | 核心功能免费，深度诊断 ¥9.9/次 |
| JobScan 中文支持差 | 中国本土化，支持互联网/金融/外企/国企风格 |
| ChatGPT 需要自己写 prompt | 一键上传，结构化报告，零门槛 |

## 技术方案

### 数据层
- **简历库**：脱敏后的简历结构化数据（教育/经历/技能/项目）
- **JD 数据库**：各行业岗位 JD，提取技能关键词、经验要求、学历要求
- **诊断结果库**：诊断报告 + 用户反馈（投递结果），用于优化算法
- **用户画像**：目标行业、求职阶段（校招/社招/跳槽）、诊断历史

### 逻辑层
```
简历输入（PDF/Word/图片）
        ↓
  OCR 识别 + 结构化提取（Tesseract / pdfplumber / 简历专用解析器）
        ↓
  5 维度诊断引擎（规则引擎 + LLM 评分）
        ↓
  JD 对标分析（关键词提取 → 向量匹配 → 差距识别）
        ↓
  改进建议生成（LLM 生成具体可执行建议 + 改后版本预览）
        ↓
  ATS 友好度检测（格式检查 + 关键词密度 + 结构评分）
```

### 展示层
- **Web 端（核心）**：上传 → 诊断 → 对比 → 导出（MVP 阶段优先）
- **微信小程序**：拍照上传简历，碎片化使用场景（Phase 2）
- **浏览器插件**：在招聘网站一键诊断（Phase 3）

### 技术栈
| 层 | 技术选型 | 说明 |
|----|---------|------|
| **前端** | Next.js 14 + TailwindCSS + shadcn/ui | SSR 利于 SEO，简历对比页用 Canvas 渲染 |
| **后端** | Python FastAPI | 简历解析库生态好（pdfplumber/python-docx） |
| **LLM** | GLM-4 / Qwen-Max | 中文理解能力强，成本 ¥0.01/千 tokens |
| **OCR** | Tesseract / PaddleOCR | PaddleOCR 中文识别更准（开源免费） |
| **简历解析** | pdfplumber + 自建规则引擎 | PDF 文本提取 + 正则匹配结构化字段 |
| **向量数据库** | Milvus Lite / Qdrant | JD 关键词向量匹配，轻量级部署 |
| **数据库** | PostgreSQL + Redis | 用户数据 + 诊断缓存 |
| **部署** | Vercel（前端）+ Railway/Fly.io（后端）| 低成本，自动扩缩 |

### 关键技术细节

**简历解析流水线**：
1. PDF → pdfplumber 提取文本 → 按段落分割
2. 图片 → PaddleOCR 识别 → 文本输出
3. Word → python-docx 提取文本
4. 正则匹配提取结构化字段：
   - 手机号：`1[3-9]\d{9}`
   - 邮箱：`\w+@\w+\.\w+`
   - 学历关键词：`本科|硕士|博士|大专`
   - 时间范围：`2023\.\d+ - \d+\.\d+`
5. LLM 二次结构化：将分段文本映射为 {教育背景[], 工作经历[], 项目经验[], 技能[]}

**JD 对标匹配算法**：
1. 从 JD 提取技能关键词（TF-IDF + LLM 辅助）
2. 计算简历关键词覆盖率（精确匹配 + 同义词扩展）
3. 向量相似度计算（Sentence-BERT 编码 → 余弦相似度）
4. 输出匹配度分数 + 缺失技能清单

**诊断 Prompt 核心模板**（约 2000 tokens）：
```
你是一位资深 HR，请诊断以下简历，从 5 个维度评分（1-10）：
1. 内容完整度 2. 数据支撑度 3. 关键词匹配 4. 表达专业度 5. ATS 友好度

简历内容：
{resume_text}

目标岗位 JD：
{jd_text}

输出格式：JSON，每个维度给出分数 + 具体问题 + 改进建议（精确到段落）。
```

## 竞争壁垒

1. **效果追踪数据飞轮**：用户标记投递结果 → 哪些诊断建议真正有效 → 优化算法。这个闭环竞品均未建立，数据越积累壁垒越高。
2. **JD 对标数据库**：积累各行业岗位 JD + 技能关键词图谱，匹配度越训练越准。
3. **行业定制 Prompt 库**：互联网 STAR 法则、金融数据导向、外企英文规范、国企格式要求 —— 每个行业需要大量真实简历样本调优。
4. **用户迁移成本**：诊断历史 + 改进记录 + 投递效果追踪 → 用户换产品 = 丢失所有数据。
5. **UGC 内容壁垒**：改前改后对比图在小红书/抖音形成品牌认知，竞品短期难以复制内容资产。
