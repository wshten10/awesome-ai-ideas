## 一、项目概述

### 1.1 产品定义
TermNest 是一款面向自由译者和本地化工作者的 AI 驱动术语管理工具，通过智能术语提取、上下文感知推荐和冲突检测，帮助语言专业人士构建个人化的动态术语库，从源头解决翻译不一致问题。

### 1.2 目标用户
- **核心用户**：自由译者（全球200万+）、独立本地化工作者
- **扩展用户**：小型翻译团队（2-10人）、翻译项目协调者
- **潜在用户**：企业内翻译部门（需要轻量级术语管理的团队）

### 1.3 核心价值主张
> 翻译质量 = 收入 + 客户留存。TermNest 让术语管理从手动记忆升级为 AI 辅助的自动化流程。

#### Chrome Extension 三层架构
1. **Content Script层**：注入主流CAT工具Web端（memoQ Cloud、Smartcat、Phrase），拦截翻译输入区域
2. **Background Service Worker**：本地术语库索引+实时匹配引擎，零网络延迟
3. **Popup UI层**：侧边栏显示术语建议+冲突警告，不打断翻译流程

### 3.2 核心技术组件

#### 3.2.1 术语提取Pipeline
```
输入 → TMX/双语Excel/SDLXLIFF
  ↓
BPE分词 → 双语对齐(FastAlign) → 术语候选提取(TF-IDF+互信息) → 人工确认
  ↓
输出 → 结构化术语库(TBX格式)，支持多客户命名空间
```

**技术选型**：
- 分词：SentencePiece (多语言BPE)
- 双语对齐：FastAlign (轻量级) / Eflomal (更精确)
- 术语提取：TF-IDF + Pointwise Mutual Information
- 格式支持：TMX 2.0, SDLXLIFF, bilingual DOCX, XLSX

#### 3.2.2 实时建议引擎
- 输入区域 contenteditable 监听，200ms debounce 触发匹配
- 前缀匹配 + Levenshtein 模糊匹配，覆盖变体（复数、时态）
- 客户术语自动切换：检测当前项目文件路径中的客户标识，加载对应术语集
- 建议排序：匹配度 x 使用频率 x 最近使用时间

#### 3.2.3 离线优先设计
- 全部术语数据 IndexedDB 本地存储，不依赖网络
- 同步机制：术语变更增量上传，冲突时时间戳优先
- 隐私关键：翻译内容不离开用户设备，仅术语条目同步到云端

### 3.3 AI 能力集成

#### 3.3.1 上下文感知推荐
- **模型**：GLM-5 / GPT-4o-mini (低延迟)
- **方式**：Few-shot prompting + 术语上下文模板
- **延迟**：<500ms (术语上下文 < 200 tokens)
- **降级策略**：无网络时回退到纯匹配引擎

#### 3.3.2 冲突检测
- **触发**：实时检测源词在不同文档中的不同译法
- **分析**：LLM判断是否为真正冲突（vs 可接受的变体）
- **建议**：提供统一方案 + 保留两种译法的理由
