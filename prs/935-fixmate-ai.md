## 1. 问题背景与用户痛点

### 1.1 家电故障：独居青年的"黑暗时刻"

在中国，独居青年人口已突破200万，每年新增家电维修需求超亿次。当一个独居年轻人面对洗衣机突然罢工、空调不制冷、燃气灶打不着火的场景时，焦虑感是全方位的。

- **恐惧未知**：不知道故障严重程度，担心维修费是天价
- **信息不对称**：维修师傅说"主板坏了要换"——是真的吗？自己无法判断
- **决策瘫痪**：修还是换？花200修一台用5年的洗衣机，还是直接买新的？
- **价格参差不齐**：维修师傅报价300合理吗？同城平均价格是多少？

### 1.2 用户画像与场景

| 用户类型 | 典型场景 | 核心需求 | 付费意愿 |
|----------|----------|----------|----------|
| 独居青年 | 洗衣机突然不转了，一脸懵 | 快速诊断，判断是否需要师傅 | 中（省维修费即赚钱） |
| 新房业主 | 新房家电陆续出问题，不知道找谁 | 保修查询+靠谱师傅推荐 | 高（新家预算充足） |
| 租房族 | 房东家电坏了，不确定责任归属 | 故障责任判定+费用预估 | 中（与房东协商依据） |
| 宝妈/家庭 | 厨房电器故障影响日常生活 | 紧急处理建议+安全判断 | 高（时间敏感） |

### 1.3 市场规模

- 中国家电保有量超30亿台，年故障率约8-12%
- 家电维修市场规模2000亿元，但数字化渗透率不足5%
- 80/90后占维修需求主体的65%，高度依赖线上信息决策

### 2.2 核心模块设计

#### 模块1: 多模态故障诊断

**输入**: 故障照片/视频 + 文字/语音描述 + 设备品牌型号

**处理流程**:
```
1. 图像预处理：设备外观 + 故障部位特写 + 铭牌信息OCR
2. 品牌型号识别：铭牌OCR + 视觉匹配 → 精确到具体型号
3. 故障现象理解：多模态LLM融合分析图像+文字描述
4. 知识库检索：RAG查找该型号常见故障+维修案例
5. 原因排序：按概率排序可能原因，附置信度
```

#### 模块2: 修or换决策引擎

**决策因素模型**:

```python
class RepairOrReplaceEngine:
    def evaluate(self, device: Device, fault: Fault, cost: CostEstimate) -> Decision:
        factors = {
            "repair_cost_ratio": cost.estimate / device.current_value,  # 维修费残值比
            "device_age_ratio": device.age / device.expected_lifespan,  # 使用年限比
            "energy_efficiency_gain": self.calc_energy_savings(device),  # 换新的省电收益
            "repair_risk": fault.recurrence_probability,  # 修后复发概率
            "sentiment_value": device.user_attachment_score,  # 情感价值（老物件）
        }
        
        score = self.weighted_score(factors)
        
        if score > 0.7:
            return Decision.REPLACE  # 建议换新
        elif score > 0.3:
            return Decision.REPAIR_IF_UNDER(cost.threshold)  # 有条件维修
        else:
            return Decision.REPAIR  # 建议维修
```

#### 模块3: DIY分步指南生成

对于可自行处理的简单故障，AI生成个性化维修教程：

```

难度: ⭐⭐ (简单，无需工具)
预计耗时: 15分钟
所需工具: 盆、毛巾、一字螺丝刀

步骤1: 断电断水 → 拔掉电源插头，关闭水龙头 [安全提醒⚠️]
步骤2: 找到排水过滤网 → 通常在前面板右下角，有个小盖板
步骤3: 打开盖子 → 下面放好盆接水，逆时针拧开过滤网
步骤4: 清理堵塞物 → 检查有无硬币、头发、纸屑等
步骤5: 装回测试 → 拧紧过滤器，通电试运行排水程序

💡 常见坑：过滤器拧不开别硬来，可能还有隐藏螺丝
📸 如果不确定位置，拍照发给我帮你确认
```

#### 模块4: 费用预估引擎

**数据来源**:
1. 众包价格数据：用户维修后匿名分享实际费用
2. 维修平台数据：啄木鸟家庭维修、万师傅等公开报价
3. 品牌官方维修价目表
4. 配件价格数据库（淘宝/1688批发价作为成本参考）

**预估逻辑**:
```
基准价 = 同城同品牌同故障历史中位数
调价因子 = {地区系数 × 紧急程度 × 楼层(是否有电梯) × 师傅等级}
预估区间 = 基准价 × 调价因子 × [0.8, 1.2]  // 20%浮动
```
