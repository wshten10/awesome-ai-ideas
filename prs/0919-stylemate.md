# StyleMate — AI 驱动的个性化着装推荐与胶囊衣橱管理助手

> **Issue:** #919 · **目标用户:** 职场新人和着装困惑者  
> **English Title:** StyleMate — AI-Powered Personal Outfit Recommendation & Capsule Wardrobe Manager

---

## 1. 概述

StyleMate 解决「每天穿什么」的决策疲劳问题。通过拍照数字化衣橱、学习个人风格偏好、结合天气与日程场景，AI 每日生成个性化穿搭方案，并智能分析衣橱缺口引导精准消费。

### 核心价值主张

- **消除决策疲劳**：早上不再站在衣柜前发呆
- **提高衣橱利用率**：从 20% 提升到 80%，减少浪费
- **建立个人风格**：避免被网红带偏，找到适合自己的穿搭体系
- **精准消费**：只买真正需要的单品，拒绝冲动购物

## 4. 功能设计

### P0 — MVP 核心功能

#### 4.1 📸 衣橱数字化
- 拍照自动识别：类别、颜色、材质、风格标签、适合场景
- 智能分类：上衣/下装/外套/鞋履/配饰
- 去重检测：提醒已有相似单品

**技术方案：** CLIP 嵌入 + 多模态 LLM 属性提取  
**冷启动策略：** 「风格测试 + 3-5张核心单品拍照」即可开始使用，不要求全量上传

#### 4.2 🌤️ 每日穿搭推荐
- 输入：天气（温度/湿度/风力）+ 日程（会议/面试/休闲/运动）
- 输出：2-3 套搭配方案 + 选择理由
- 用户反馈：喜欢/不喜欢/替换某件 → 持续优化

**技术方案：** 约束求解（天气+场景+衣橱可用性）+ LLM 创意微调

#### 4.3 🎯 风格测试
- 10 道视觉选择题，建立初始风格画像
- 维度：极简/商务/休闲/文艺/街头/运动
- 输出风格雷达图 + 推荐风格方向

### P1 — 增长功能

#### 4.4 🛒 智能购物顾问
- 拍照/截图想买的单品 → AI 判断与衣橱兼容度（0-100分）
- 「衣橱缺口分析」：你有5件黑T恤但缺休闲裤
- 价格区间建议，避免过度消费

#### 4.5 📊 穿搭统计
- 衣橱利用率、单品穿着频率排行
- 「吃灰」单品预警
- 月度穿搭回顾

### P2 — 生态功能

#### 4.6 📅 胶囊衣橱方案
- X 件单品搭配 Y 套 look 的数学优化
- 换季衣橱轮换提醒
- 极简主义衣橱挑战

#### 4.7 👥 社交功能
- 朋友互看衣橱，互相推荐
- 「帮我参谋」分享链接
- 穿搭打卡社区

### 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 衣物识别 | CLIP + 多模态LLM | CLIP做视觉嵌入，LLM做属性提取，互补 |
| 穿搭推荐 | 约束求解 + LLM | 约束保证可行性，LLM注入创意 |
| 风格学习 | 交互反馈 → 风格向量微调 | 无需大量训练数据，用户行为即训练 |
| 移动端 | React Native | 跨平台，拍照为核心交互 |

### MVP 简化策略

根据 Review 建议，MVP 聚焦「拍照录入 + 场景匹配」，跳过复杂的搭配推荐算法：
1. 风格测试（10题选择题）
2. 拍照录入3-5件核心单品
3. 基于天气+日程的简单搭配推荐
4. 喜欢/不喜欢反馈

## 8. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| **冷启动**：用户不愿上传全量衣橱 | 高 | 高 | 「风格测试+少量拍照」组合降低门槛，首日即可获得推荐 |
| **推荐质量**：初期推荐不够精准 | 中 | 高 | 先用规则引擎兜底，用户反馈数据积累后切换AI推荐 |
| **拍照识别准确率**：叠放/褶皱衣物识别困难 | 中 | 中 | 支持手动标签修正，迭代优化分割模型 |
| **竞争**：小红书等平台加AI推荐 | 中 | 中 | 核心壁垒在个人真实衣橱数据，平台只有泛化推荐 |
| **隐私**：衣橱照片涉及个人信息 | 低 | 高 | 端侧处理优先，照片提取属性后不上传原图 |

## 10. 总结

StyleMate 切入「每天穿什么」这个高频低门槛的决策场景，通过「个人真实衣橱数字化 × AI场景推荐 × 精准消费引导」的三位一体方案，区别于现有衣橱管理工具和泛化穿搭推荐平台。

**核心优势：**
- 冷启动门槛低：风格测试 + 3-5张照片即可使用
- 场景化推荐：天气+日程+个人风格的三维匹配
- 胶囊衣橱理念：「买得少穿得好」，契合可持续消费趋势
- 商业路径清晰：免费增值 + 品牌联营，ARPU有提升空间

**MVP 验证的关键假设：** 用户是否愿意为「每天不费脑子穿得好看」这件事持续付费？
