# AI 浏览器代理调度器 — 智能网页任务编排系统

> **一句话卖点**：用 YAML 写一个网页自动化任务，调度器自动分配 Agent 执行，失败时 AI 视觉理解智能重试，比 UiPath 便宜 50 倍，比 n8n 更懂网页

## 概述

基于 Issue #153 创建。当前网页自动化市场呈现爆发态势：browser-use (84.4k⭐)、firecrawl (98.0k⭐)、n8n (181.0k⭐)、langflow (146.2k⭐) 的 GitHub 热度证明需求真实存在。但现有方案存在三个核心断层：

1. **单 Agent 限制**：browser-use 只能单次操作，无法编排「登录 → 导航 → 提取 → 通知」多步骤任务
2. **无智能容错**：网页结构一变，选择器就废，任务直接失败
3. **高门槛高成本**：传统 RPA（UiPath/Blue Prism）年费数万，n8n 学习曲线陡峭

本产品通过「声明式任务 DSL + 多 Agent 并行调度 + AI 视觉容错 + 任务市场」四个维度解决上述痛点。产品经理评估 9/10，技术专家可行性 7/10，商业分析 7/10。

**核心价值**：不是又一个浏览器自动化工具，而是网页任务的**编排操作系统**。

## 功能设计

### 核心功能
1. **任务 DSL**：YAML/JSON 声明式定义多步骤网页任务，支持条件分支、循环、变量传递
2. **多 Agent 调度**：任务分析 → Agent 匹配 → 资源分配 → 并行执行，支持优先级队列
3. **智能重试**：选择器失效时 AI 视觉理解（截图 → VLM 定位元素）自动尝试备选方案
4. **会话管理**：Cookie 持久化 + 多账号隔离，避免重复认证
5. **任务市场**：社区分享/复用自动化流程，版本控制 + 评价体系
6. **可视化编排器**：拖拽生成 YAML，非技术用户也能用

### 差异化优势
| 现有方案 | 本产品 |
|---------|--------|
| browser-use 单次操作 | 多 Agent 并行编排，复杂任务自动分解 |
| n8n 拖拽 20 个节点 | YAML 10 行搞定同样任务 |
| UiPath ¥50,000/年 | 开源免费，云服务 ¥99/月起 |
| Selenium 写代码 | 声明式 YAML，零代码门槛 |
| 选择器一变就废 | AI 视觉理解 + 多重备选策略 |
| 单账号串行 | 多账号隔离 + 优先级并行 |
| 无社区模板 | 任务市场，一键复用社区流程 |

## 技术方案

### 三层架构

#### 数据层
- **任务模板库**：YAML/JSON 模板 + 版本管理（类 npm registry）
- **执行日志**：每步操作截图 + DOM 快照 + AI 决策日志
- **用户凭证**：AES-256 加密存储，可选 HashiCorp Vault 集成
- **选择器索引**：网站 → 选择器映射，社区维护
- **会话池**：Cookie/Token 持久化，TTL 自动过期

#### 逻辑层
```
用户提交 YAML
    ↓
┌─────────────────┐
│  任务解析器       │  YAML → AST → 执行计划
│  (Parser)        │  校验语法 + 依赖检查
└────────┬────────┘
         ↓
┌─────────────────┐
│  调度器          │  任务分析 → Agent 匹配 → 资源分配
│  (Scheduler)     │  优先级队列 (BullMQ/Redis)
│                  │  并行度控制 + 超时管理
└────────┬────────┘
         ↓
┌─────────────────┐
│  执行引擎        │  每个 Agent 独立 Browser Context
│  (Executor)      │  Playwright 实例池管理
│                  │  选择器 → DOM 操作
└────────┬────────┘
         ↓
┌─────────────────┐
│  容错层          │  选择器失效 → AI 视觉定位
│  (Fallback)      │  截图 → VLM → 坐标映射
│                  │  指数退避 + 异常分类
└────────┬────────┘
         ↓
    结果输出 / 通知
```

**Agent 通信协议**（技术专家建议用 MCP）：
- 基于 JSON-RPC 2.0 的轻量协议
- 支持上下文共享（共享 Cookie/Session）
- 竞态检测：两个 Agent 同时操作同一网站时自动排队
- 超时隔离：单 Agent 崩溃不影响其他任务

#### 展示层
- **CLI**：`browser-scheduler run task.yaml` — 开发者首选
- **Web UI**：任务管理 + 执行监控 + 可视化编排器
- **API**：RESTful + WebSocket（实时推送执行状态）
- **VS Code 插件**：YAML 语法高亮 + 本地调试

### 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| 浏览器引擎 | **Playwright** | 跨浏览器、API 现代、社区活跃 |
| 任务队列 | **BullMQ + Redis** | 优先级队列、延迟任务、成熟稳定 |
| 任务编排 | **Temporal**（Phase 3） | 长时间任务编排、断点续传、可视化管理 |
| AI 视觉 | **GPT-4o / Qwen-VL** | 页面截图 → 元素定位，成本 $0.01/次 |
| 后端框架 | **Node.js + Fastify** | 高性能、TypeScript 原生、与 Playwright 同生态 |
| 数据库 | **PostgreSQL** | 任务日志 + 用户数据，JSONB 存 YAML 模板 |
| 缓存 | **Redis** | 任务队列 + 会话缓存 + 速率限制 |
| 容器化 | **Docker** | 每个 Agent 独立容器，崩溃隔离 |
| 前端 | **React + Monaco Editor** | YAML 编辑器 + 任务仪表盘 |

### 反爬对抗策略池
- **请求层**：随机延迟 2-8s、User-Agent 轮换（100+ 池）、代理池轮换
- **指纹层**：Playwright stealth 插件、Canvas/WebGL 指纹随机化
- **行为层**：模拟人类操作轨迹（鼠标移动、滚动、随机停顿）
- **检测层**：识别验证码 → AI 识别（hCaptcha/reCAPTCHA）或人工兜底
