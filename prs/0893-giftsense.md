## 1. 项目概述与痛点量化

### 1.1 项目愿景

GiftSense 旨在通过 AI 深度理解收礼人的兴趣、性格、生活阶段与情感需求，为送礼者提供**真正个性化**的礼物推荐方案。不同于现有平台的"热门榜单"或简单标签匹配，GiftSense 构建了收礼人的多维画像模型，结合场景感知、预算约束和情感表达，生成包含理由说明的完整送礼方案。

**核心价值主张**: 让每一份礼物都"懂"收礼人，而非仅仅"匹配"关键词。

### 1.2 目标市场与用户画像

#### 用户画像 A: 小王 - 异地恋青年 (25岁, 程序员)

- **场景**: 女友在另一个城市，每月一次小惊喜 + 生日/纪念日大礼
- **痛点**: "我不知道她最近喜欢什么，问多了显得不关心，不问又怕买错"
- **期望**: 系统能记住女友偏好历史，推荐有惊喜感的礼物
- **月预算**: ¥500-2000

#### 用户画像 B: 李经理 - 职场商务送礼 (35岁, 销售总监)

- **场景**: 客户维护、节日商务礼、新合作伙伴初次见面礼
- **痛点**: "商务送礼最怕不合时宜，太贵对方不敢收，太便宜又失礼"
- **期望**: 场景化的得体推荐，考虑文化禁忌和商务礼仪
- **次均预算**: ¥200-2000（视关系层级浮动）

#### 用户画像 C: 张阿姨 - 异地父母 (55岁, 退休教师)

- **场景**: 给在外地工作的子女/孙辈寄礼物
- **痛点**: "不知道年轻人现在流行什么，网上推荐的东西看着都一样"
- **期望**: 语音输入偏好，操作简单，推荐实用且有温度的礼物
- **月预算**: ¥200-800

#### 用户画像 D: 陈同学 - 理工思维人群 (22岁, 研究生)

- **场景**: 给朋友/恋人/家人选礼物，追求理性决策
- **痛点**: "我需要数据和逻辑支撑，不想被营销文案忽悠"
- **期望**: 看到推荐理由、性价比分析、用户真实评价摘要
- **月预算**: ¥100-500

### 1.4 核心痛点量化

1. **信息不对称 (78%用户反馈)**: 送礼者对收礼人当前需求和偏好了解不足
2. **选择过载 (65%用户反馈)**: 电商平台上"礼物"关键词返回10万+结果，筛选成本极高
3. **预算焦虑 (52%用户反馈)**: 不知道同等预算下什么是"好"的选择
4. **形式化严重 (44%用户反馈)**: 送礼变成任务而非情感表达，缺乏创意和心意
5. **售后无追踪 (38%用户反馈)**: 不知道对方是否喜欢，无法形成反馈闭环

### 2.3 AI模型策略

#### 主推理模型 (Primary LLM)

- **GPT-4o**: 多轮对话管理、自然语言理解、用户意图识别
- **温度设置**: 对话模式 0.7（创意性），推荐模式 0.3（确定性）
- **上下文窗口**: 128K上下文注入完整画像+历史对话+商品信息
- **结构化输出**: 强制JSON Schema输出，确保推荐结果可解析

#### 推理增强模型 (Reasoning LLM)

- **Claude 3.5 Sonnet**: 收礼人画像推理、场景分析、推荐理由生成
- **优势**: 更擅长复杂推理链和多因素权衡分析
- **触发条件**: GPT-4o推荐置信度低于阈值时二次验证

#### 降级策略 (Fallback Chain)

```
GPT-4o -> Claude 3.5 Sonnet -> GPT-4o-mini -> 本地Qwen2.5-7B -> 规则引擎兜底
```

- P99延迟: < 3秒（对话），< 8秒（完整推荐）
- 降级触发: API超时(>10s)、429速率限制、503不可用

#### 嵌入模型

- **文本嵌入**: text-embedding-3-large (3072维)
- **图像嵌入**: CLIP ViT-L/14 (参考图片搜索)
- **向量索引**: IVF_PQ，召回率>95%，延迟<50ms

### 2.4 数据流设计

```
用户输入(文字/图片/语音)
  -> NLU: 意图分类 + 实体提取
  -> 画像查询: PostgreSQL加载收礼人画像
  -> 场景匹配: Drools规则引擎
  -> 知识图谱: Neo4j扩展相关品类
  -> 商品检索: ES精确过滤 + Milvus语义召回
  -> LLM推理: 生成3-5个推荐+理由+心意建议
  -> 响应组装: JSON -> 前端渲染
```

### 3.2 模块二: 场景感知推荐引擎 (Context-Aware Recommendation Engine)

#### 场景分类体系

- **birthday**: 同辈/长辈/儿童生日，预算¥100-2000，避免送钟伞等谐音不吉物品
- **anniversary**: 恋爱/结婚/相识纪念日，预算¥200-5000，重视仪式感和纪念意义
- **business**: 初次见面/客户维护/节日问候，预算¥100-3000，注意企业合规
- **holiday**: 春节/中秋/情人节/母亲节等，预算¥100-3000，遵循节日传统
- **comfort**: 生病慰问/失恋安慰，预算¥50-500，避免喜庆包装和颜色
- **daily_surprise**: 无理由小礼/犒劳奖励，预算¥20-300，轻量趣味不造成压力

#### 推荐算法 Pipeline

```
Step 1: 场景解析 -> 场景分类 + 情感基调 + 紧急程度 + 文化约束
Step 2: 画像-场景交叉匹配 -> 候选品类集合，排除禁忌品类和价格外品类
Step 3: 商品检索与排序
  Phase A: Elasticsearch精确过滤 (价格、品类、品牌、评分)
  Phase B: Milvus语义相似度召回 (Top-100)
  Phase C: 多因素融合排序
    score = 0.35 * semantic_sim
          + 0.25 * profile_match
          + 0.20 * quality_score
          + 0.10 * value_score
          + 0.10 * novelty_score
Step 4: LLM推荐方案生成
  输入: Top-20商品 + 画像 + 场景 + 预算
  输出: 3-5个精选推荐，每个包含商品信息、推荐理由、心意表达建议、组合建议
```

### 3.3 模块三: 预算智能优化引擎 (Budget Optimization Engine)

#### 优化模型

```
目标函数: max Sum(w_i * satisfaction_i(x_i))
约束条件: Sum(price(x_i)) <= budget * (1 + flexibility)
其中: satisfaction_i(x) = f(profile_match, quality, novelty, emotional_value)
```

#### 预算分配策略

| 预算区间 | 推荐策略 | 示例 |
|----------|----------|------|
| ¥0-50 | 单品精选，强调心意和创意 | 手写明信片+小零食礼盒 |
| ¥50-200 | 主品+小配搭 | 精选咖啡豆+手冲杯 |
| ¥200-500 | 主品+配件+包装 | 品牌香薰+扩散器+礼盒包装 |
| ¥500-2000 | 主题礼盒/体验型礼物 | 周末酒店体验券+手工巧克力 |
| ¥2000+ | 定制化方案+体验升级 | 定制皮具+高级红酒+私人贺卡 |

#### 价格趋势监控

- 接入电商平台价格API，监控推荐商品价格波动
- 大促前提醒: "该商品历史最低价在618，建议等待"
- 价格保护: 推荐商品7天内降价则通知用户

### 3.4 模块四: 礼物记忆库 (Gift Memory Vault)

#### 数据模型

```sql
CREATE TABLE gift_records (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES users(id),
    recipient_id    UUID NOT NULL REFERENCES recipients(id),
    scene           VARCHAR(50) NOT NULL,
    scene_detail    JSONB,
    budget          DECIMAL(10,2),
    actual_spent    DECIMAL(10,2),
    recommended_items JSONB NOT NULL,
    selected_item   JSONB,
    purchase_link   TEXT,
    status          VARCHAR(20) DEFAULT 'recommended',
    -- recommended / purchased / delivered / feedback_collected
    recipient_feedback JSONB,
    -- { rating: 1-5, comment: "...", emotion_tags: [...] }
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    feedback_at     TIMESTAMPTZ,
    ai_notes        TEXT
);

CREATE INDEX idx_gift_user_recipient ON gift_records(user_id, recipient_id);
CREATE INDEX idx_gift_created ON gift_records(created_at DESC);
```

#### 功能特性

- **时间线视图**: 按时间轴展示所有送礼记录，标注重要节点
- **AI复盘**: 每次收集反馈后，AI自动分析"什么有效、什么可以改进"
- **重复检测**: 警告"您去年送过类似的礼物给同一个人"
- **关系升温曲线**: 可视化送礼频率和反馈趋势，反映关系变化
- **年度报告**: 年终生成"您的送礼年度总结"，包含数据洞察

### 3.5 模块五: 心意表达增强 (Emotional Expression Enhancer)

#### 生成策略

```
输入维度:
  - 收礼人画像 (性格、关系深度、沟通风格)
  - 场景 (生日/纪念日/日常/商务)
  - 礼物类型 (实用型/情感型/体验型)
  - 用户表达风格 (正式/随意/幽默/感性)
  - 关键信息 (共同回忆、内部梗、特殊意义)

输出:
  - 贺卡文案 (3个版本: 温馨版/幽默版/深情版)
  - 送礼时可以说的话 (口头表达建议)
  - 包装建议 (颜色、风格、附加小物件)
```

#### 风格适配示例

| 收礼人类型 | 推荐风格 | 示例文案 |
|------------|----------|----------|
| INFP文艺型 | 诗意含蓄 | "想到你的时候，世界变得很轻" |

| ENTP幽默型 | 轻松有趣 | "别问我为什么选这个，问就是直觉(其实是AI推荐的)" |
| 长辈传统型 | 温暖恭敬 | "一份心意，愿您身体健康，万事顺遂" |

#### 商务场景特殊处理

- 礼品卡/红包的合规文案模板
- 企业logo定制礼品的搭配建议
- 文化差异提醒（国际商务场景）

| **AI能力** | GPT-4o+Claude双模型 | 推荐算法 | 协同过滤 | 规则引擎+ML | 无 |
| **预算优化** | 5/5 智能组合 | 1/5 无 | 2/5 价格筛选 | 3/5 预算过滤 | 1/5 无 |
| **反馈闭环** | 5/5 收礼人反馈 | 2/5 点赞收藏 | 2/5 评价系统 | 2/5 无 | 1/5 无 |
| **心意表达** | 5/5 AI生成文案 | 2/5 用户自写 | 1/5 无 | 2/5 模板 | 2/5 编辑文案 |
| **目标市场** | 中国 | 中国 | 中国 | 北美/欧洲 | 中国 |
| **商业模式** | 免费+SaaS | 广告+电商CPS | 电商交易佣金 | 订阅制 | 广告 |

### 4.2 竞品详细分析

#### 竞品1: 小红书礼物推荐

- **优势**: UGC内容丰富，社区氛围强，年轻用户基数大
- **劣势**: 推荐基于浏览记录非收礼人画像，商业推广混杂，无法针对特定人推荐
- **机会**: GiftSense可以整合小红书内容作为评价数据源

#### 竞品2: 淘宝/京东礼物频道

- **优势**: 商品库庞大，交易闭环完整，物流体系成熟
- **劣势**: 标签体系粗放（"送女友""送长辈"），缺乏个性化推理能力
- **机会**: 作为电商API合作伙伴，GiftSense提供"AI导购"增值服务

#### 竞品3: GiftAdvisor (海外)

- **优势**: 北美市场先行者，问卷式画像采集有参考价值
- **劣势**: 缺乏中国送礼文化适配（禁忌、礼仪），无LLM推理能力
- **机会**: GiftSense的文化适配和AI推理形成差异化壁垒

#### 竞品4: 礼物说 (已关停 - 教训参考)

- **失败原因**: 过度依赖编辑团队，内容生产成本高，商业模式单一
- **教训**: 纯内容模式不可持续，需要技术驱动的规模化能力
- **启示**: GiftSense的AI驱动模式天然具备规模化优势

### 4.3 差异化优势总结

1. **唯一以收礼人为中心的AI画像系统** - 非商品为中心
2. **双LLM推理架构** - GPT对话+Claude推理，互补优势
3. **文化深度适配** - 中国送礼礼仪、禁忌、潜规则的系统化编码
4. **反馈闭环** - 从推荐到送达再到收礼人反馈的完整数据链
5. **情感计算** - 不止推荐商品，更关注送礼的情感表达

## 5. 商业模式

### 5.1 收费模式

#### Free 免费版

#### Pro 专业版 (19.9元/月 或 168元/年)

- 无限AI推荐
- 最多10个收礼人画像
- 全场景支持（含商务送礼）
- 预算智能优化
- 礼物记忆库完整功能
- 心意表达增强（AI贺卡文案）
- 优先客服支持
- **目标**: 核心付费用户群，预计转化率8-12%

- Pro全部功能
- 无限收礼人画像
- 社交信号分析（授权微博/朋友圈）
- 定制化礼物方案（AI+人工顾问）
- 企业批量送礼管理
- API接口（企业集成）
- 价格趋势监控和提醒
- **目标**: 高价值用户和企业客户
