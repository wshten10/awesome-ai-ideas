# AI 预约管家 - 智能预约管理助手

> **一句话卖点**：健忘族在收到预约确认时，用 AI 预约管家截图识别，从错过重要预约损失 ¥200+ 到提前3天自动提醒，再也不怕预约撞车

## 概述

基于 Issue #111 创建的详细设计文档。

现代人面临「预约碎片化」困境：医疗预约（医院、牙医、体检）、生活服务（美容、理发、修车）、社交活动（聚餐、会议、约会）分散在微信、美团、短信、邮件等多个平台。错过预约不仅浪费钱（取消费用、重新排队），更造成时间损失和心理焦虑。

AI 预约管家通过**多模态智能识别 + 统一时间线 + 冲突检测**，帮助用户告别「错过预约」的焦虑，实现零操作的自动化预约管理。

## 功能设计

### 核心功能

1. **多模态智能识别**
   - 截图识别：拍照/截图自动提取预约信息（时间、地点、服务内容）
   - 短信解析：Android 自动识别预约确认短信（需权限授权）；iOS 手动转发短信到应用
   - 邮件解析：读取预约确认邮件（需 IMAP 授权）
   - 置信度分层：高置信度自动录入，低置信度人工确认
   - **平台限制说明**：iOS 严格限制短信读取权限，建议用户使用「截图识别」或「手动转发」作为主要方式

2. **统一时间线视图**
   - 所有预约在一个时间线上展示
   - 分类视图：医疗 / 生活服务 / 社交 / 个人成长
   - 心理时间感：今天 / 明天 / 本周 / 未来

3. **智能提醒系统**
   - 多级提醒：提前 3 天 / 1 天 / 2 小时
   - 动态调整：根据预约类型调整提醒时间（牙医提前 1 天，飞机提前 3 天）
   - 个性化学习：根据用户历史行为优化提醒时机

4. **冲突检测与预警**
   - 时间重叠检测
   - 地理位置分析（两个预约之间交通时间够不够？）
   - 准备时间建议（就诊需要提前到，理发不需要）
   - 身体状态考虑（牙医和按摩排太近可能不是好主意）

5. **一键改约助手**
   - 生成礼貌、得体的改期/取消话术
   - 提供最佳改期时间建议（避开高峰）
   - 一键复制话术到剪贴板（用户自行粘贴到微信/短信/邮件）
   - **说明**：由于微信未开放 API，无法实现一键发送，采用复制到剪贴板的方式

### 增值功能

- **快速导入**：从系统日历导入历史预约、从常用平台导出数据（解决冷启动问题）
- **家人共享**：管理家人的预约（孩子的疫苗、父母的体检）
- **优先级标识**：用颜色/图标区分重要程度
- **历史统计**：分析用户的预约习惯和错过率

## 技术方案

### 数据层

**数据最小化原则**：只提取预约相关的必要字段（时间、地点、服务内容），不存储原始短信/邮件内容，保护用户隐私。

```
┌─────────────────────────────────────────┐
│           数据采集层                      │
├─────────────────────────────────────────┤
│ 截图识别  │  短信解析  │  邮件解析        │
│ (OCR)     │  (规则引擎)│  (IMAP+NER)     │
│           │  [Android] │  [需授权]        │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           结构化数据存储                  │
├─────────────────────────────────────────┤
│ • 预约时间、地点、服务内容                │
│ • 来源平台、置信度                       │
│ • 用户确认状态                           │
└─────────────────────────────────────────┘
```

### 逻辑层

```
┌─────────────────────────────────────────┐
│           AI 分析引擎                    │
├─────────────────────────────────────────┤
│ • 预约分类（医疗/生活服务/社交）          │
│ • 冲突检测算法                           │
│ • 智能提醒时机计算                       │
│ • 话术生成（改期/取消）                  │
└─────────────────────────────────────────┘
```

### 展示层

```
┌─────────────────────────────────────────┐
│           用户界面                       │
├─────────────────────────────────────────┤
│ • 统一时间线视图                         │
│ • 冲突预警提示                           │
│ • 智能提醒通知                           │
│ • 一键改约操作                           │
└─────────────────────────────────────────┘
```

### 技术栈建议

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 截图识别 | PaddleOCR / GLM-4V | 中文识别准确率高 |
| 短信解析 | 规则引擎 + NER | Android 自动识别；iOS 手动转发/截图 |
| 邮件解析 | IMAP + LLM | 结构化信息提取 |
| 冲突检测 | 规则引擎 + LLM 混合 | 多维度判断 |
| 话术生成 | GLM-4 / Claude | 生成礼貌、得体的话术 |
| 提醒系统 | 本地通知 + 云端推送 | 确保不遗漏 |
| 数据存储 | SQLite (本地) + 云端同步 | 隐私友好 |

## 风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| **隐私合规风险** - 读取短信/邮件涉及敏感数据 | 高 | 中 | • 明确的隐私政策<br>• 本地优先处理策略<br>• 用户授权机制<br>• 数据加密存储<br>• **数据最小化原则**：只提取必要字段（时间、地点、服务），不存储原始短信/邮件内容<br>• **合规说明**：符合 GDPR 和中国《个人信息保护法》要求，提供数据导出/删除功能 |
| **平台封闭性** - 微信小程序数据难获取 | 高 | 高 | • 先做短信+邮件识别<br>• 截图识别作为主要入口<br>• 引导用户手动上传截图 |
| **识别准确率不足** - 不同平台格式差异大 | 中 | 中 | • 置信度分层策略<br>• 人工确认机制<br>• 持续优化识别模型 |
| **冷启动问题** - 用户不愿主动截图所有预约 | 中 | 高 | • 从短信/邮件自动识别开始<br>• 提供价值证明（错过预约成本计算）<br>• 游戏化激励（连续记录奖励）<br>• **快速导入功能**：支持从系统日历导入历史预约、从常用平台（美团、大众点评）导出数据 |
| **竞争压力** - 微信/支付宝可能推出类似功能 | 高 | 低 | • 专注细分场景（医疗预约）<br>• 建立数据壁垒<br>• 快速迭代保持领先 |

## 差异化优势

### vs. 日历应用

| 传统日历 | AI 预约管家 |
|---------|-----------|
| 手动添加，操作繁琐 | 拍照/截图自动识别，零操作 |
| 当天提醒，来不及调整 | 提前 3 天提醒，充足准备 |
| 无冲突检测 | 多维度冲突预警 |
| 无改期辅助 | 一键生成改期话术 |

### vs. 平台自带提醒

| 平台提醒 | AI 预约管家 |
|---------|-----------|
| 分散在各平台，难以统筹 | 统一时间线，一目了然 |
| 提醒时间固定 | 智能调整提醒时机 |
| 无冲突检测 | 自动发现预约撞车 |
| 无改期辅助 | 提供最佳改期建议 |
