# AI PR 自动摘要与变更影响分析

> 用 AI 自动分析 PR，生成摘要、分类变更、分析影响范围，让 Code Review 更高效

## 功能设计

### 核心功能
- **智能摘要**：一键生成 PR 做了什么、为什么改的清晰摘要
- **变更分类**：自动识别新功能 / Bug 修复 / 重构 / 文档 / 测试
- **影响分析**：识别可能受影响的模块、API、依赖项
- **测试建议**：根据变更内容推荐重点测试场景

### 增强功能
- **风险评级**：高/中/低风险标记，优先关注关键变更
- **关联追踪**：自动关联相关 Issue、PR、文档
- **团队协作**：Review 意见聚合，变更讨论串

## 技术方案

### 数据层
- **GitHub API**：获取 PR diff、commits、files changed
- **向量数据库**：存储代码库上下文，支持语义检索
- **缓存层**：已分析 PR 结果缓存，避免重复计算

### 逻辑层
- **Diff 解析器**：解析 unified diff，提取变更块
- **LLM 分析引擎**：
  - 变更意图理解
  - 影响范围推断
  - 测试场景生成
- **规则引擎**：基于模式的风险评估

### 展示层
- **GitHub App**：PR 评论区自动生成报告
- **Web Dashboard**：团队级 PR 分析统计
- **IDE 插件**：VS Code 中查看 PR 分析

### Phase 3: 企业版 (4 周)

## 资源需求

### API
- **LLM API**：Claude / GPT-4（代码理解能力强）
- **GitHub API**：PR 数据获取
- **Embedding API**：代码向量检索

### 服务
- **托管**：Vercel / Cloudflare Workers（Serverless）
- **数据库**：PostgreSQL + pgvector
- **队列**：Redis / Bull（异步任务）

### 成本估算
- MVP 阶段：~$50/月（API 调用）
- 规模化后：按 PR 计费，边际成本递减

## 差异化策略

1. **深度影响分析**：不仅说改了什么，更说会影响什么
2. **垂直化**：先聚焦 React/Next.js 生态，建立专家形象
3. **可解释性**：每个结论都给出推理过程
4. **团队协作**：Review 效率统计，团队知识沉淀

## 风险与对策

| 风险 | 对策 |
|------|------|
| Copilot 碾压 | 聚焦差异化功能（影响分析） |
| LLM 成本高 | 缓存 + 增量分析 + 小模型辅助 |
| 准确性问题 | 人工反馈 + 持续优化 prompt |
