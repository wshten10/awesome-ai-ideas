# AI 会议健康守护者

> 长时间会议导致疲劳？智能守护你的会议健康，自动提醒休息、会后恢复建议

## 功能设计

### 核心功能
1. **实时会议监测**
   - 集成 Google Calendar / Outlook / 飞书日历
   - 追踪连续会议时长、会议密度
   - 识别"会议马拉松"（3+ 小时无间隔）

2. **智能休息提醒**
   - 检测会议间隙过短（<10 分钟）时提前预警
   - 建议插入休息时段，可一键添加到日历
   - 支持自定义休息间隔偏好

3. **会后恢复建议**
   - 自动生成 2 分钟微运动（伸展、眼保健操）
   - 根据会议类型推荐恢复活动（脑力会→散步，体力会→拉伸）
   - 番茄钟式专注恢复计时器

4. **周度健康报告**
   - 统计会议时长分布、连续会议次数
   - 对比团队/行业平均值
   - 生成个性化优化建议

## 技术方案

### 数据层
- **日历集成**：Google Calendar API / Microsoft Graph API / 飞书开放平台
- **本地存储**：SQLite 存储用户偏好和历史数据
- **隐私保护**：会议内容不上传，仅分析时间元数据

### 逻辑层
- **规则引擎**：基于规则判断会议密度（连续 3 场 1h+ = 高密度）
- **提醒调度**：本地推送通知（macOS / Windows / Linux）
- **建议生成**：预设恢复动作库 + LLM 个性化推荐

### 展示层
- **桌面应用**：Electron / Tauri 跨平台
- **系统托盘**：后台运行，非侵入式提醒
- **Web Dashboard**：周报查看和偏好设置

## 实现步骤

### Phase 1: MVP（2 周）
1. Google Calendar 单一集成
2. 基础连续会议检测（3+ 场无 15 分钟间隔）
3. 系统通知提醒
4. 3 个预设恢复动作

### Phase 2: 增强版（4 周）
1. 多日历平台支持（Outlook、飞书）
2. 一键插入休息时段
3. 恢复动作库扩展（20+ 动作）
4. 周度报告生成

### Phase 3: 智能版（6 周）
1. LLM 个性化建议
2. 团队会议健康分析（B2B）
3. 与健康设备联动（Apple Watch 久坐提醒）

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 日历隐私敏感 | 仅读取时间元数据，不上传会议内容 |
| 多平台集成复杂 | MVP 先做单平台，验证需求再扩展 |
| 提醒疲劳 | 可调节提醒频率，支持"专注模式"静默 |
