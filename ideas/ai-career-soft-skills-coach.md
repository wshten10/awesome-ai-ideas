# AI 职场软技能导师 - 新人的贴身成长教练

> **一句话卖点**：职场新人在会议前 10 分钟，用 AI 导师生成发言要点和问答预案，从不敢开口到自信发言

## 概述

基于 Issue #71 创建的详细设计文档。

职场新人（0-3年）最大的挑战往往不是技术能力，而是软技能：会议中不敢发言、收到模糊需求不知道如何澄清、向上汇报不知道该说什么、与同事/客户沟通常常词不达意。这些技能学校不教，同事不一定有时间教，自己摸索成本高。

**AI 职场软技能导师** 是一个针对职场场景的智能教练，通过场景模拟、邮件优化、会议准备、复盘分析等功能，帮助职场新人快速提升软技能。

## 功能设计

### 核心功能

1. **沟通模拟训练**
   - 场景选择：汇报、反馈、谈判、冲突、面试、绩效面谈
   - 角色扮演：AI 扮演上级/同事/客户，用户练习对话策略
   - 实时反馈：分析用户回应，给出更好的表达建议
   - 语音模式：支持语音对话，更接近真实场景

2. **邮件/消息优化**
   - 粘贴草稿，AI 分析语气、结构、清晰度
   - 一键改写：更专业/更委婉/更直接
   - 场景适配：汇报、请求、反馈、道歉等模板

3. **会议准备助手**
   - 输入会议主题和自己的角色
   - 生成：发言要点、可能被问到的问题及回答建议
   - 时间控制：根据会议时长生成精简/完整版

4. **场景复盘分析**
   - 描述遇到的情况和自己的处理方式
   - AI 分析：哪里做得好、哪里可以改进、更好的处理方式
   - 积累个人案例库，追踪成长轨迹

### 用户流程

```
新用户注册 → 选择岗位和痛点 → 推荐学习路径 →
场景模拟训练 → 获得反馈 → 实际应用 → 复盘优化
```

## 技术方案

### 数据层

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 用户数据 | PostgreSQL | 用户画像、学习进度、案例库 |
| 场景库 | PostgreSQL + Redis | 预设场景模板，高频缓存 |
| 对话记录 | PostgreSQL | 训练历史，用于个性化 |
| 文件存储 | S3/R2 | 语音文件、学习报告 |

### 逻辑层

| 服务 | 技术选型 | 说明 |
|------|----------|------|
| API Gateway | Node.js/Fastify | 统一入口，鉴权，限流 |
| 对话引擎 | LLM API (GLM-4/Claude) | 场景模拟、反馈生成 |
| 语音服务 | Whisper ASR + Edge TTS | 语音识别和合成 |
| 分析引擎 | LLM API | 邮件优化、复盘分析 |
| 推荐系统 | 规则引擎 + 向量检索 | 个性化场景推荐 |

### 展示层

| 端 | 技术选型 | 说明 |
|------|----------|------|
| 移动端 | React Native/Flutter | 随时随地练习 |
| Web 端 | React/Vue | 深度学习和报告查看 |
| 微信小程序 | 原生/Taro | 快速触达用户 |

## 实现步骤

### Phase 1: MVP (4 周)

**目标**：验证核心价值 - 场景模拟训练

| 周 | 任务 | 交付物 |
|----|------|--------|
| W1 | 场景库设计 + 5 个核心场景 | 场景模板文档 |
| W2 | 对话引擎 + 文字模拟 | 文字版场景模拟 |
| W3 | 反馈生成 + 用户测试 | 反馈算法 + 测试报告 |
| W4 | Web 端基础界面 | 可用的 MVP |

**MVP 范围**：
- 5 个高频场景（汇报、反馈、谈判、冲突、面试）
- 文字对话模拟
- 基础反馈（做对/做错/建议）
- 简单用户系统

### Phase 2: 功能扩展 (4 周)

- 邮件/消息优化
- 会议准备助手
- 语音模式（可选）
- 移动端适配

### Phase 3: 个性化 (4 周)

- 学习路径推荐
- 个人案例库
- 成长报告
- 企业版基础功能

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 场景库冷启动 | 高 | 先聚焦 5 个高频场景，逐步扩展；用户生成内容补充 |
| 用户付费意愿不确定 | 高 | MVP 免费，用数据验证需求；先做 B2B 验证 |
| 语音延迟影响体验 | 中 | 优先做文字版，语音作为高级功能；使用流式 TTS |
| 场景模拟真实性不足 | 中 | 持续优化 Prompt；引入真实案例；用户反馈迭代 |
| 竞品模仿 | 中 | 持续积累场景库和用户数据；建立品牌壁垒 |
| 内容合规风险 | 低 | 场景设计避免敏感话题；用户协议明确责任边界 |
