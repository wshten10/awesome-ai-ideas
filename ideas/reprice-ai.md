# RePrice AI — 二手物品智能估价与最优出售策略助手

> **一句话卖点**：拍张照片，3秒知道你的闲置值多少钱、该挂哪个平台、怎么写文案卖最快

## 概述

基于 Issue #923 创建的详细设计文档。

中国二手交易市场规模超万亿，但卖家面临三大难题：**不会定价**（定高了无人问津，定低了血亏）、**懒得写描述**（拍完照就搁置）、**不知道挂哪**（闲鱼/转转/小红书各有门道）。**RePrice AI** 让用户拍一张照片，即可获得 AI 估价、吸引人的文案和多平台最优发布策略。

## 功能设计

### 核心功能

1. **拍照即估价**
   - 拍摄物品照片，AI 识别品牌、型号、成色
   - 结合市场实时数据给出建议价格区间（最低价/建议价/最高价）
   - 展示同类物品近期成交价分布图
   - 支持多角度照片提高估价精度

2. **智能文案生成**
   - 根据物品特征自动生成吸引人的出售标题（含热门关键词）
   - 生成带情感共鸣的商品描述（如"毕业清仓""搬家急出"）
   - SEO 优化提高搜索曝光率
   - 一键复制到闲鱼/转转/小红书

3. **最优发布策略**
   - 分析各平台（闲鱼、转转、小红书）同类物品流量规律
   - 推荐最佳发布时间段（如闲鱼工作日午休、小红书晚间）
   - 建议定价策略（一口价/可刀/拍卖）
   - 标签推荐提高搜索命中率

4. **成色智能评估**
   - 图像识别磨损程度：划痕、褪色、变形
   - 自动分级（全新/99新/95新/9成新/8成新/7成新）
   - 生成成色说明文案（如"轻微使用痕迹，功能完好"）
   - 用户可手动微调成色等级

5. **批量处理模式**
   - 搬家/断舍离场景：一次拍摄多件物品
   - 批量生成商品卡片（标题+描述+估价+标签）
   - 按预估价值排序，优先处理高价值物品
   - 导出为各平台兼容格式

6. **价格跟踪与调价**
   - 已发布物品未售出时，动态调价建议
   - 追踪同类成交价走势
   - 降价提醒（如"同类物品本周成交价下降15%，建议调至¥XX"）
   - 超期未售自动建议调整策略（换平台/降价/重新描述）

## 技术方案

### 数据层

```
用户数据
├── 物品库 (items)
│   ├── 物品ID、照片URL（多角度）
│   ├── 品牌、型号、类别、成色等级
│   ├── AI估价、用户定价、最终成交价
│   ├── 发布平台、发布时间、状态
│   └── 浏览量、收藏量、询问量
├── 价格数据库 (prices)
│   ├── 品牌型号 → 历史成交价分布
│   ├── 成色等级 → 价格折损系数
│   ├── 平台 → 流量/转化率数据
│   └── 季节性价格波动
└── 文案模板库 (templates)
    ├── 高转化标题模板
    ├── 成色描述话术
    └── 标签/关键词库
```

### 逻辑层

1. **图像识别服务**
   - 多模态 LLM（GPT-4o / GLM-4V）：识别品牌、型号、类别
   - 成色评估：磨损检测 + 自动分级
   - 调用方式：用户上传 → 多模态分析 → 结构化输出

2. **估价引擎**
   - 众包价格数据：社区用户上报成交价（解决反爬问题）
   - 官方残值模型：品牌官方以旧换新价格作为锚点
   - 回归模型：品牌 × 型号 × 成色 → 价格区间
   - 冷启动：初始用品牌指导价 + 折旧公式

3. **文案生成**
   - LLM 生成个性化描述（输入：物品属性 + 成色 + 平台风格）
   - A/B 测试框架：不同文案版本追踪转化率
   - 热词库更新：爬取各平台高流量商品标题关键词

4. **平台策略分析**
   - 各平台流量数据（公开统计 + 社区众包）
   - 发布时间优化模型
   - 定价策略推荐算法

### 展示层

- **移动端优先**：微信小程序 / React Native App
- **拍照流程**：引导式多角度拍摄（正面/背面/细节/瑕疵）
- **估价卡片**：价格区间 + 成交价分布图 + 同款在售数量
- **一键发布**：生成文案后直接跳转闲鱼/转转发布页
- **仪表盘**：在售物品状态、调价提醒、成交统计

## 资源需求

### API
- **多模态 LLM**：GPT-4o / GLM-4V（物品识别 + 成色评估）
- **LLM**：GPT-4o-mini / GLM-4-Flash（文案生成）
- **图片存储**：AWS S3 / Cloudflare R2

### 服务
- **数据库**：PostgreSQL / Supabase（物品 + 价格数据）
- **向量数据库**：用于相似物品检索（Milvus / Qdrant）
- **消息推送**：调价提醒（FCM / APNs）

### 预估成本（月活 1 万）
- 多模态 API：$400-800（图片识别 + 成色评估）
- 文案生成 API：$100-200
- 存储和数据库：$100-150
- 总计：$600-1150/月

## 变现模式

### Freemium 模式

- **免费版**：
  - 每月 10 次拍照估价
  - 基础文案生成
  - 单平台发布策略

- **Pro 版（¥15/月）**：
  - 无限拍照估价
  - 多平台文案适配
  - 价格跟踪 + 调价提醒
  - 批量处理模式
  - 成交数据分析

### 增长策略
1. **「测测你的闲置值多少钱」** 免费工具引流（社交裂变）
2. **搬家/断舍离季** 定向营销（毕业季、年末大扫除）
3. **社区成交价上报** 激励机制（上报成交价获 Pro 时长）
4. **KOL 合作**：家居博主/断舍离博主推荐

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 闲鱼/转转无公开 API，价格数据难获取 | 1. 众包价格数据（用户上报成交价换积分）<br>2. 品牌官方以旧换新价格作为锚点<br>3. 避免直接爬取，降低法律风险 |
| 成色评估主观性强，AI 精度不足 | 1. 用户可手动微调成色等级<br>2. 多角度拍摄提高评估精度<br>3. 积累标注数据持续优化模型 |
| 多模态 API 成本高 | 1. 先识别类别+品牌，再用文本模型估价<br>2. 建立本地缓存，相同型号不重复调用<br>3. 限速免费用户使用量 |
| 用户冷启动动力不足 | 1. 「测测闲置值多少」零门槛体验<br>2. 首次估价结果可一键分享社交平台<br>3. 搬家场景批量处理是强拉动 |
| 自动发布功能平台封控 | 1. MVP 先做「一键复制文案」，不做自动发布<br>2. 后期探索平台合作接入 |

## 成功指标

- **估价准确率**：AI 估价与实际成交价偏差 < 20%
- **文案采纳率**：用户使用 AI 文案 > 70%
- **付费转化率**：免费 → Pro 转化 > 5%
- **月活留存**：30 日留存 > 35%（非一次性工具，靠调价提醒+新物品持续回来）
