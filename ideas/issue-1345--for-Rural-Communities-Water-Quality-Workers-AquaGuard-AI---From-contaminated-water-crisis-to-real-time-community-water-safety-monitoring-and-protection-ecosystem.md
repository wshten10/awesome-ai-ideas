## 🎯 问题背景与用户痛点

中国农村地区有超过5亿人口依赖分散式供水（井水、小型水厂、山泉水），饮用水安全长期缺乏有效监管。根据生态环境部数据，2023年全国农村水源地超标率仍达12.7%，涉及重金属、农药残留、微生物污染等多种危害。

**四大核心痛点：**

1. **突发性污染事件发现滞后**：农村水源地常发生农药泄漏、生活污水直排、工业废水偷排等突发性污染，传统依赖人工巡检的方式往往在污染已经造成健康损害后才被发现。2024年甘肃某村砷超标事件中，从污染发生到发现间隔长达3个月，导致全村23人中毒。

2. **水质检测成本高、周期长**：传统实验室检测单次费用300-800元，出结果需3-7天。农村小型水厂年营收仅5-20万元，无力承担日常检测费用，大部分村庄每季度甚至半年才检测一次。

3. **数据孤岛严重**：环保局、水利局、卫健委各自管理水质数据，缺乏统一的预警机制。村民无法及时获取水质信息，出了问题不知道找谁、怎么处理。

4. **应急响应机制缺失**：污染事件发生后，缺乏智能化的应急预案和资源配置建议。村干部不知道应该通知谁、是否需要停水、临时安全水源在哪里。

### 目标用户画像

|------|------|---------|
| **农村居民**（老人、儿童为主） | 日常饮水安全 | 可靠的水质信息，污染时及时通知，安全饮水方案 |
| **村委/水厂管理员** | 水源日常管理 | 低成本监测工具，简单易用，异常自动告警 |

| **县级疾控中心** | 公共卫生监测 | 水质数据对接健康监测，疾病预警联动 |

## 🤖 AI技术方案

### 核心架构

```
IoT传感层 → 边缘计算网关 → 云端AI引擎 → 多端应用
    ↓              ↓              ↓            ↓
 水质参数     本地预处理      AI预警分析    村民小程序
 (pH/浊度/    异常初筛       多源数据融合   管理后台
  余氯/温度)   断网缓存       知识图谱      预警推送
```

### 🛠️ 核心AI能力

#### 1. 多模态数据融合分析
- **传感数据**：pH值、浊度、余氯、溶解氧、温度、电导率（低成本传感器，单价<200元）
- **气象数据**：降雨量、气温、风速（对接气象局API，影响地表水扩散）
- **历史数据**：该水源地过去3年水质监测记录、污染事件档案
- **地理数据**：周边污染源位置（养殖场、化工厂、农田）、地形、水系流向

#### 2. LSTM时序异常检测模型
```
输入：过去72小时传感器时序数据 + 气象数据 + 历史基线
模型：双向LSTM + Attention机制
输出：未来24小时各指标预测值 + 异常概率评分

训练数据来源：
- 国家水质监测网公开数据（2000+站点，5年历史）
- 农村饮水安全工程监测数据（合作获取）
- 自建试点站点积累数据
```

#### 3. 污染溯源AI引擎
- 当检测到异常时，基于地理信息系统 + 风向/水流模型，自动推断可能的污染源
- 结合知识图谱（污染类型→可能来源→检测指标映射），生成污染溯源报告
- 示例：pH骤降 + 硝酸盐升高 → 上游农田化肥径流可能性80%

#### 4. 应急响应知识图谱
```
污染类型 → 健康风险等级 → 应对措施 → 临时水源方案 → 通知对象
  │
  ├── 微生物超标 → 中等风险 → 煮沸饮用 + 消毒片 → 邻村安全水源 → 疾控中心
  ├── 重金属超标 → 高风险 → 立即停水 + 瓶装水 → 应急送水 → 环保局+医院
  ├── 农药残留   → 高风险 → 停水 + 活性炭过滤 → 送水 + 临时取水点 → 农业局
  └── 浊度异常   → 低风险  → 沉淀过滤 → 暂不需要 → 自行处理
```

### 🛠️ 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **边缘计算** | 树莓派4B + 定制Python服务 | 成本<500元/节点，社区支持好 |

| **前端** | 微信小程序（村民端）+ Vue3（管理后台） | 微信渗透率农村>70% |
| **消息推送** | 微信服务通知 + 语音电话（紧急） | 确保老人也能收到告警 |
| **地图服务** | 高德地图API | 农村覆盖度好，中文POI丰富 |

### AI模型训练计划

1. **数据收集阶段（1-2个月）**：
   - 对接国家水质监测网公开数据（2000+站点×5年）
   - 与省级环保厅合作获取农村水源监测数据
   - 在试点站点积累本地数据（至少3个月连续数据）

2. **模型训练阶段（1个月）**：
   - 基础模型：双向LSTM + Attention，输入窗口72小时
   - 异常检测：Isolation Forest + LSTM预测残差，双模型交叉验证
   - 溯源模型：基于图神经网络的污染传播模拟

3. **模型部署与优化（持续）**：
   - 边缘侧：ONNX格式，推理延迟<100ms，支持离线运行
   - 云端：完整模型，每周重新训练，月度A/B测试
   - 冷启动：先用规则引擎（阈值+变化率），积累数据后切换AI

## 💰 商业模式

### 市场规模分析

中国农村饮水安全监测市场当前渗透率<2%，存在巨大的增长空间：

- **直接市场**：全国约60万个行政村，按每村年均监测服务费2000元计算，市场空间约12亿元/年
- **延伸市场**：农业用水监测、养殖场排水监测、小型食品厂进水监测等，额外市场空间约8亿元/年
- **政府市场**：县级/省级水质监测平台建设，市场空间约15亿元/年
- **合计可触达市场**：约35亿元/年，年增长率15-20%

### 收入模式

**1. 政府采购（主要收入，占70%）**
- 县级农村饮水安全监测平台：20-50万元/年（含硬件+服务）
- 省级饮用水安全态势感知系统：100-300万元/项目
- 应急监测服务：5-10万元/次（突发污染事件后快速部署）

**2. B2B订阅（占20%）**
- 村级水厂监测套餐：500元/月（含2个传感器+平台+告警）
- 养殖场排水监测：800元/月（排污合规监管）
- 小型食品厂进水监测：600元/月（原料水安全）

**3. 增值服务（占10%）**
- 水质检测报告生成：200-500元/份
- 污染溯源分析报告：2000-5000元/份
- 农业保险数据服务（为保险公司提供水质风险评估数据）

1. **力合科技（300203.SZ）**：上市公司，专注城市水质监测，单站造价>20万元。农村市场覆盖几乎为零。
2. **碧水源（300070.SZ）**：水务工程为主，监测为附属业务。产品线偏重，不适合分散式水源。
3. **小型IoT创业公司**：如"水滴监测"，功能单一（仅传感采集），缺乏AI分析和预警能力。
4. **政府自建系统**：部分省份建了监测平台，但传感器覆盖率<5%，农村基本空白。

## 📊 数据模型设计

```sql
-- 水源站点表
CREATE TABLE water_sources (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),           -- 水源名称（如"张家村水井"）
    village VARCHAR(100),        -- 所属村庄
    source_type VARCHAR(20),     -- 类型：井水/地表水/小型水厂
    latitude DECIMAL(10,7),
    longitude DECIMAL(10,7),
    installed_at TIMESTAMP,
    status VARCHAR(20) DEFAULT 'active'
);

-- 传感器读数表（时序数据）
CREATE TABLE sensor_readings (
    id BIGSERIAL PRIMARY KEY,
    source_id INT REFERENCES water_sources(id),
    recorded_at TIMESTAMP DEFAULT NOW(),
    ph_value DECIMAL(4,2),       -- pH值
    turbidity DECIMAL(6,2),      -- 浊度(NTU)
    residual_chlorine DECIMAL(4,2), -- 余氯(mg/L)
    dissolved_oxygen DECIMAL(5,2),  -- 溶解氧(mg/L)
    temperature DECIMAL(4,1),    -- 温度(℃)
    conductivity DECIMAL(8,2),   -- 电导率(μS/cm)
    nitrate DECIMAL(6,2),        -- 硝酸盐(mg/L) -- 高级传感器
    is_anomaly BOOLEAN DEFAULT FALSE,
    anomaly_score DECIMAL(3,2),  -- AI异常概率(0-1)
    raw_data JSONB               -- 原始传感器数据
);

-- 预警记录表
CREATE TABLE alerts (
    id SERIAL PRIMARY KEY,
    source_id INT REFERENCES water_sources(id),
    alert_level VARCHAR(10),     -- info/warning/critical
    alert_type VARCHAR(50),      -- ph_anomaly/turbidity_spike/...
    description TEXT,
    triggered_at TIMESTAMP DEFAULT NOW(),
    resolved_at TIMESTAMP,
    resolution TEXT,
    notified_users JSONB         -- 已通知的用户列表
);
```

## 🏛️ 政策与合规

### 相关政策红利

1. **《农村饮用水安全巩固提升工程实施方案》（2024-2026）**：中央财政每年投入200亿元用于农村饮水安全，IoT监测设备采购纳入补贴范围。
2. **《数字乡村发展行动计划》**：推动农村地区数字化基础设施建设，水质在线监测系统被列为重点建设内容。
3. **《生活饮用水卫生标准》（GB 5749-2022）**：新国标增加嗅味、铝、消毒副产物等指标，对监测系统提出更高要求，利好智能化解决方案。
4. **乡村振兴专项资金**：各省设立专项，可用于采购农村水质监测设备和服务。

### 合规要求

- 水质数据需符合《生态环境监测网络建设方案》数据标准
- 预警系统需与当地疾控中心建立数据共享机制
- 传感器设备需通过计量认证（CMA）
- 数据存储需满足《数据安全法》和《个人信息保护法》要求

## 💡 创新亮点

1. **"哨兵模式"低成本部署**：不需要全指标高端传感器，用5个基础指标（pH/浊度/余氯/温度/电导率）的组合异常检测，就能覆盖90%以上的常见水质问题。单站点成本控制在2000元以内。

2. **AI预测 vs 传统阈值**：传统方案是"超标了才报警"，AquaGuard AI是"趋势异常就预警"。LSTM模型能在指标仍在安全范围内时，通过时序变化趋势提前24小时预测可能的超标事件。

3. **知识图谱驱动的应急响应**：不是简单报警，而是自动生成应对方案。pH骤降→可能是酸雨或工业废水→建议停水+送检→推荐临时水源。让村干部知道"该做什么"。

4. **"最后一公里"信息触达**：水质信息通过微信服务通知、语音电话、村委广播三种渠道触达村民，确保不使用智能手机的老人也能收到告警。
