## 🎯 How to Contribute

### Submit a New Idea

| Your idea's maturity | Submit to | Notes |
|---------------------|-----------|-------|
| 💡 Early concept | [`proposals/`](./proposals) | A few paragraphs describing the core idea |
| 🟢 Well-formed idea | [`ideas/`](./ideas) | Follow the [template](./TEMPLATE.md) |
| ⭐ Complete design | PR to [`pr/`](./pr) | Full tech specs and business analysis |

### Other Ways to Participate

- **Discuss ideas** — Have thoughts on an existing idea? [Open an Issue](../../issues/new)
- **Claim & build** — Want to bring an idea to life? Comment on the idea document
- **Improve** — Found something to optimize? Submit a PR directly

### Idea Document Structure

Each idea should include these sections. See [`TEMPLATE.md`](./TEMPLATE.md) for details:

```
Overview → Feature Design → Technical Plan → Implementation → Resources → Monetization → Competitive Analysis → Risks
```
