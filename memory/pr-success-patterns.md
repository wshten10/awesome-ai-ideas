## High-Quality PR Patterns (Rating 8+)

### Content Structure (300+ lines recommended)
1. **项目背景** - Market context with specific data points (% of users, market size)
2. **问题现状** - 5+ pain points with quantified impact
3. **2026年关键机遇** - Policy, technology, market timing
4. **技术方案** - Architecture diagrams + core modules
5. **实施路线图** - Phase 1/2/3 with specific milestones
6. **商业模式** - Pricing tiers, revenue streams, unit economics
7. **竞争分析** - 4+ named competitors with comparison table
8. **风险评估** - High/Medium/Low with mitigation strategies
9. **目标用户画像** - Demographics, behavior, pain points
10. **社区讨论总结** - Credit contributors, show consensus

### Key Differentiators
- **Data-backed claims**: "年均减产15-30%" not just "产量下降"
- **Specific competitors**: Name 4+ actual products with URLs
- **Unit economics**: CAC, LTV, LTV/CAC ratio, gross margin
- **Risk classification**: High/Medium/Low with probability and mitigation
- **Architecture diagrams**: ASCII art showing layers
- **Implementation phases**: MVP → Growth → Ecosystem with timelines

### PR Title Format
- `feat: [Product Name] - [Chinese subtitle describing value] (Issue #XXX)`
- Example: `feat: AI智能农业助手 - 从经验种植到数据驱动的精准农业 (Issue #389)`

### PR Body vs File Content
- Best PRs put full content in a **separate file** (e.g., `pr/ai-agriculture-assistant-389.md`)
- PR body serves as summary with link to full content
- This allows much longer, more detailed content

### Low-Quality Patterns (Rating <6, to avoid)
- Short PR body (under 100 lines) with no separate file
- Missing competitive analysis
- Vague market claims without data
- No pricing or business model details
- No risk assessment
- Generic technology descriptions
- Missing user personas
