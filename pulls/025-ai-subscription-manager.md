# AI 订阅管家 - 智能追踪你的数字支出

> 自动发现沉睡订阅，帮你找回「消失的钱」

## 功能设计

### 核心功能
1. **支出透视** - 自动扫描银行账单/邮件，汇总所有订阅费用
2. **使用分析** - 检测哪些服务你 actually 在用，哪些闲置
3. **重复检测** - 发现功能重叠的订阅（如多个音乐 App）
4. **智能建议** - 推荐取消/降级方案，计算每月可省金额

### 差异化优势
- **本地优先**：敏感数据不上云，隐私有保障
- **多源聚合**：邮件 + 账单 + API 多渠道获取数据
- **AI 洞察**：不只是统计，还能发现隐藏的订阅模式

## 技术方案

### 数据层
- **邮件扫描**：IMAP 连接 Gmail/Outlook，NER 提取订阅确认信
- **账单解析**：OCR（Tesseract/PaddleOCR）+ 规则引擎识别订阅扣费
- **本地存储**：SQLite 加密存储，用户完全掌控数据

### 逻辑层
- **订阅识别**：命名实体识别（NER）+ 规则匹配
- **使用频率追踪**：可选 API 对接（Spotify/Netflix 等）或手动标记
- **智能建议引擎**：基于规则 + LLM 生成个性化取消建议

### 展示层
- **Dashboard**：月度支出趋势、订阅分布饼图
- **订阅清单**：按类别/价格/使用频率排序
- **省钱报告**：「本月发现 3 个沉睡订阅，可省 ¥128」

## 实现步骤

### Phase 1: MVP（2 周）
1. 邮件扫描：支持 Gmail OAuth + 订阅确认信识别
2. 手动导入：PDF/截图账单 OCR 解析
3. 基础 Dashboard：订阅列表 + 月度总支出
4. 本地优先：所有数据处理在本地完成

### Phase 2: 智能化（2 周）
5. 使用频率追踪：API 对接主流服务（Spotify/Netflix/Apple）
6. 重复检测：识别功能重叠的订阅
7. 智能建议：LLM 生成个性化取消建议

### Phase 3: 增值（2 周）
8. 省钱报告：每周/月推送订阅健康报告
9. 家庭版：多账户合并统计
10. 降价提醒：监测订阅涨价并通知

## 资源需求

### API & 服务
- **Gmail API**：OAuth 邮件访问（免费额度充足）
- **OCR 服务**：PaddleOCR 本地部署（无 API 成本）
- **LLM**：建议生成（可选，可用本地模型）

### 技术栈
- **前端**：React + Recharts（可视化）
- **后端**：Node.js / Python FastAPI
- **本地存储**：SQLite + sqlcipher（加密）
- **部署**：桌面应用（Electron/Tauri）或 本地 Web

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 银行数据获取难 | 先做邮件扫描 + 手动导入，验证价值后再对接银行 |
| 隐私顾虑 | 强调本地处理，提供「仅本地模式」 |
| 用户不愿持续使用 | 省钱报告提供持续价值，推送发现新沉睡订阅 |

## 成功指标

- **用户月均省钱**：≥ ¥50（验证真实价值）
- **沉睡订阅发现率**：≥ 80% 用户发现至少 1 个
- **持续使用率**：30 日留存 ≥ 40%
