# JusticeKit AI - AI驱动的自助诉讼导航与文书生成助手

> **一句话卖点**：让普通人也能在法庭上有效维权，AI生成法律文书+全流程导航

## 概述

### 问题背景

美国每年超过300万起民事案件中当事人没有律师代理（pro se），胜诉率不到10%。80%的低收入美国人法律需求未被满足。核心痛点：

- **法律文书门槛极高**：起诉状、答辩状有严格格式要求，写错可能直接被驳回
- **诉讼流程如同迷宫**：立案→送达→举证→开庭→判决→执行，每个环节都有法定期限
- **庭审准备严重不足**：不知道如何回答法官提问、组织证据链、进行交叉询问
- **法律信息获取不平等**：富人花几千美元咨询律师，低收入者只能搜索碎片化信息

> 一位单亲妈妈面临非法驱逐，因为写不好答辩状错过提交期限，带着两个孩子流落街头。她本有充足的法律理由可以抗辩，却输在了"不会写"和"不知道期限"上。

### 解决方案

JusticeKit AI 为无法负担律师费用的自诉当事人提供：
1. **法律文书生成引擎** — AI生成符合管辖地格式要求的文书草稿
2. **诉讼流程导航** — 个性化时间线、待办清单和期限提醒
3. **模拟庭审问答** — 基于案件材料模拟法官提问
4. **证据链分析** — 识别证据薄弱环节，建议补充方向

### 目标用户

- **主要用户**：低收入人群、中小企业主、租客、消费者维权者
- **次要用户**：法律援助组织、公益律师
- **典型场景**：租客面临非法驱逐、消费者买到假货在小额法庭起诉、小企业合同纠纷、离婚财产分割
- **痛点强度**：🔴 高频刚需 — 涉及财产、自由、基本权利

## 技术方案

### 核心架构

```
┌──────────────┐    ┌───────────────┐    ┌───────────────┐    ┌──────────────┐
│  引导式问卷   │ →  │  RAG检索引擎   │ →  │  LLM文书生成   │ →  │  合规验证层   │
│  收集案件信息 │    │  法律条文+案例  │    │  GPT-4/Claude │    │  律师审核标记  │
└──────────────┘    └───────────────┘    └───────────────┘    └──────────────┘
         ↕                 ↕                    ↕                    ↕
┌──────────────┐    ┌───────────────┐    ┌───────────────┐    ┌──────────────┐
│  诉讼状态机   │    │  法律知识图谱  │    │  案例相似匹配  │    │  风险分级系统  │
│  期限提醒     │    │  实时同步更新  │    │  胜诉率预测    │    │  高风险→人工   │
└──────────────┘    └───────────────┘    └───────────────┘    └──────────────┘
```

### 关键AI能力

| 能力 | 技术实现 | 说明 |
|------|---------|------|
| 文书生成 | GPT-4/Claude + RAG | 结合本地法规数据库生成文书 |
| 流程导航 | 状态机模型 | 按管辖地提供个性化时间线 |
| 模拟庭审 | LLM角色扮演 | 模拟法官提问，预演回答 |
| 证据分析 | LLM + 规则引擎 | 识别证据链薄弱环节 |
| 案例匹配 | 嵌入向量 + 知识图谱 | 引用类似案例判决结果 |
| 多语言 | LLM原生能力 | 服务移民和少数族裔群体 |

### 安全边界

- 明确标注**不构成法律意见，不替代律师**
- 涉及刑事案件**严格引导**用户寻求专业律师
- 不自动向法院提交任何文件，必须由用户审核确认
- 高风险文件**强制律师审核**

### 竞品分析

| 竞品 | 定位 | 局限 | JusticeKit差异 |

### 差异化壁垒

2. **Jurisdiction感知** — 根据用户所在地具体法规生成文书
3. **全流程覆盖** — 从发现问题到案件结束的完整导航
4. **多语言+文化适配** — 服务移民、少数族裔等最弱势群体
5. **风险分级系统** — 低风险AI处理，高风险人工介入
