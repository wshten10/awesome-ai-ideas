# AI 知识产品化助手 - 详细设计方案

> **一句话卖点**：专业领域从业者在周末，用 AI 知识产品化助手，通过对话式引导，7 天把脑袋里的专业经验变成可售卖的完整课程

## 概述

基于 Issue #88 创建的详细设计文档。

**核心价值**：帮助有丰富经验但不会做课程的专业人士，通过 AI 对话式引导，快速将隐性知识显性化、结构化，最终产出可售卖的知识产品。

**目标用户**：
- 工作 3-10 年的专业人士（设计师、程序员、运营、HR、财务等）
- 有经验积累，但缺乏课程制作能力
- 想要知识变现，但不知道从何开始
- 周末时间有限，希望快速上手

**用户场景**：
> 周五晚上 11 点，做了 8 年 UX 设计的小林盯着电脑发呆。她积累了一堆项目经验、设计模板、踩坑笔记，想做一门课变现，但不知道从何开始。这时候，她打开了 AI 知识产品化助手...
>
> 系统问她：「你最常被请教的问题是什么？」小林随口说了几个。AI 帮她梳理出 3 个可产品化的方向，并生成了课程大纲初稿。「原来我的经验这么值钱！」

## 功能设计

### 核心功能

#### 1. 对话式知识提取
- **引导性问题系统**：根据用户专业领域，动态生成针对性问题
  - 「你最常被请教的问题是什么？」
  - 「你解决过最棘手的案例是什么？」
  - 「新手最容易踩的坑有哪些？」
- **渐进式对话**：根据回答动态调整后续问题，避免审问感
- **隐性知识显性化**：将用户零散的经验片段提取出来

#### 2. 智能大纲生成
- **结构化课程框架**：自动生成章节、课时、知识点
- **难度分层**：支持初级/中级/高级版本
- **时长估算**：自动估算每个章节的学习时长
- **可编辑**：用户可以调整顺序、增删内容

#### 3. 内容模板填充
- **章节模板库**：
  - 概念讲解模板
  - 案例分析模板
  - 实战演练模板
  - 作业与答疑模板
- **AI 辅助写作**：根据大纲自动生成初稿
- **风格一致性**：保持全篇语言风格统一

#### 4. 竞品分析与定价策略
- **竞品扫描**：分析同领域课程的价格、销量、评价
- **差异化定位**：找出用户课程的独特卖点
- **定价建议**：基于竞品分析给出合理价格区间
- **营销文案生成**：生成吸引目标用户的课程介绍

#### 5. 多格式输出
- **电子书**：PDF/EPUB 格式
- **视频脚本**：包含镜头分解、时长、配图建议
- **社群内容**：21 天打卡计划、社群讨论话题
- **营销素材**：海报文案、朋友圈推广文案

### 辅助功能

#### 6. 30 秒快速体验
- 用户输入专业话题，AI 立即生成超简短大纲（3 个要点）
- 让用户快速感受「我的经验可以这样整理」的惊喜感

#### 7. 课程共创（V2）
- 多个专业人士的互补知识自动拼接成完整课程
- 例如：「前端开发」+「UI 设计」+「产品思维」=「独立开发者全流程」

## 技术方案

### 数据层

#### 知识图谱
- **领域知识库**：预构建各专业领域的知识图谱
  - 设计领域：UX/UI、交互设计、用户研究...
  - 技术领域：前端/后端/算法/运维...
  - 运营领域：内容运营/用户运营/增长...
- **问题模板库**：每个领域的引导性问题库
- **课程模板库**：优秀课程的结构模板

#### 用户数据
- **对话记录**：用户的所有回答和历史对话
- **课程草稿**：自动保存的中间版本
- **竞品数据**：爬取的各平台课程信息

### 逻辑层

#### 对话引擎
```
技术栈：GPT-4/Claude API + LangChain
核心流程：
1. 领域识别（用户输入 → NER 分类）
2. 动态问题生成（基于领域 + 用户回答历史）
3. 知识提取（从对话中提取关键经验）
4. 结构化存储（存入知识图谱）
```

#### 大纲生成器
```
技术栈：GPT-4 + 规则引擎
核心算法：
1. 知识聚类（将零散知识点分组）
2. 难度评估（判断每个知识点的学习难度）
3. 路径规划（生成最优学习顺序）
4. 时长估算（基于知识点数量和难度）
```

#### 竞品分析器
```
技术栈：爬虫 + NLP + 数据可视化
数据源：
- 得到/知乎/小鹅通的课程信息
- 价格、销量、评价、目录结构
- 用户评论的情感分析
```

### 展示层

#### Web 界面
- **技术栈**：React + Next.js + Tailwind CSS
- **核心页面**：
  - 对话页面（类似 ChatGPT）
  - 大纲编辑器（拖拽调整）
  - 内容编辑器（Markdown + 实时预览）
  - 竞品分析看板（可视化图表）

#### 移动端
- **技术栈**：React Native 或 PWA
- **核心场景**：随时记录灵感、查看进度

## 资源需求

### API 服务
- **LLM API**：GPT-4/Claude（对话、大纲生成、内容生成）
  - 预估成本：¥0.5-1 元/用户/次完整使用
- **TTS API**（V2）：视频脚本语音版
  - 预估成本：¥0.1 元/分钟
- **存储服务**：阿里云 OSS/腾讯云 COS
  - 预估成本：¥100-500/月（初期）

### 人力需求
- **全栈开发**：1 人（MVP）
- **产品设计**：0.5 人（兼职或外包）
- **领域专家**：5 人（兼职，每个领域 1 人，负责知识图谱构建）

### 服务器
- **初期**：阿里云/腾讯云轻量服务器（¥200/月）
- **扩展后**：需要独立数据库服务器（¥1000/月）

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| **课程质量参差不齐**损害口碑 | 1. 建立课程质量评分机制<br>2. 低分课程不推荐<br>3. 提供「专家审核」增值服务 |
| **用户可能不愿持续付费** | 1. 降低订阅价格（¥49/月 而非 ¥99/月）<br>2. 提供一次性买断选项（¥499）<br>3. 增加课程共创功能，提升粘性 |
| **AI 生成内容有「AI 腔」** | 1. 大量 prompt 调优<br>2. 提供「风格选择」（正式/轻松/幽默）<br>3. 支持用户上传自己的文案样本 |
| **竞品模仿** | 1. 快速建立领域知识图谱壁垒<br>2. 积累用户对话数据，优化模型<br>3. 持续迭代新功能（课程共创、多格式输出） |
| **知识付费市场趋于饱和** | 1. 精准定位「腰部专业人士」（非头部讲师）<br>2. 聚焦「7 天快速出课」卖点<br>3. 提供 B2B 企业培训场景 |
| **用户隐私顾虑**（对话内容涉及商业机密） | 1. 明确数据使用政策<br>2. 提供本地化部署选项（企业版）<br>3. 数据加密存储 |
