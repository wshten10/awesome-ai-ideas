# AI 宠物健康日志分析师 - 从日常记录发现潜在问题

> 通过智能日志记录和 AI 分析，帮助宠物主人及早发现毛孩子的健康问题

## 功能设计

### 核心功能
- **📝 智能日志记录**
  - 多模态输入：语音、文字、图片
  - 结构化模板：进食、排便、活动、精神状态
  - 快捷标签：一键记录常见行为

- **📈 趋势分析引擎**
  - 自动识别异常模式（饮水量突增、食欲下降）
  - 个体基准线建立（每只宠物有自己的正常范围）
  - 跨指标关联分析（活动量减少 + 食欲下降 = 可能生病）

- **⚠️ 智能预警系统**
  - 分级预警：关注级、警告级、紧急级
  - 上下文感知：考虑年龄、品种、历史健康记录
  - 可配置阈值：用户可调整敏感度

- **🏥 就诊辅助**
  - 自动生成健康报告（时间线、趋势图、异常点）
  - 宠物档案导出（PDF 格式，方便就医）
  - 症状追踪记录（帮助兽医诊断）

- **💡 个性化养护建议**
  - 基于品种的养护指南
  - 年龄适配的健康提醒（疫苗、驱虫）
  - 行为解读（猫咪为什么突然躲起来？）

## 技术方案

### 数据层
```
本地优先架构：
- SQLite（本地日志存储）
- 加密存储敏感数据
- 可选云端同步（用户控制）
```

**数据模型：**
```typescript
interface PetProfile {
  id: string;
  name: string;
  species: 'cat' | 'dog' | 'other';
  breed: string;
  birthDate: Date;
  weight: WeightRecord[];
}

interface HealthLog {
  id: string;
  petId: string;
  timestamp: Date;
  category: 'food' | 'water' | 'activity' | 'behavior' | 'other';
  content: string;
  attachments?: string[]; // 图片 URL
  aiAnalysis?: AnalysisResult;
}

interface AnalysisResult {
  sentiment: 'normal' | 'attention' | 'warning' | 'urgent';
  insights: string[];
  relatedSymptoms?: string[];
}
```

### 逻辑层

**1. 日志解析器**
- NLP 提取关键信息（食量、时间、行为）
- 多模态融合（文字 + 图片分析）
- 结构化存储

**2. 异常检测引擎**
- 时序分析：滑动窗口统计（7天、30天）
- 个体基准线：基于历史数据建立正常范围
- 规则引擎 + ML 混合：
  - 规则：已知异常模式（如饮水量 > 正常 2 倍）
  - ML：发现隐藏模式（异常行为组合）

**3. 知识库系统**
- 宠物医疗知识图谱（品种常见病、症状关联）
- 本地 Embedding 检索（隐私优先）
- 定期更新机制

**4. 预警决策树**
```
输入：日志记录 → 
  解析关键指标 → 
    对比个体基准线 → 
      触发规则/ML 模型 → 
        计算风险等级 → 
          生成预警 + 建议
```

### 展示层

**移动端优先（React Native）**
- 首页：快速记录入口 + 今日概览
- 趋势页：可视化图表（进食/活动/体重曲线）
- 预警页：分级预警列表 + 处理状态
- 档案页：宠物信息 + 健康报告导出

**桌面端（Electron）**
- 深度分析仪表盘
- 批量导入历史记录
- 导出完整健康档案

## 风险与缓解

### 1. 责任边界风险
- **问题**：健康建议可能被误解为医疗诊断
- **缓解**：
  - 明确免责声明（非医疗建议）
  - 预警措辞谨慎（"建议咨询兽医"而非"可能患病"）
  - 不提供具体疾病诊断

### 2. 误报率风险
- **问题**：频繁误报会导致用户流失
- **缓解**：
  - 可调节预警敏感度
  - 用户反馈机制（标记误报，优化模型）
  - 个体基准线学习（减少误判）

### 3. 冷启动问题
- **问题**：新用户缺乏历史数据，分析质量低
- **缓解**：
  - 品种基准线（同品种宠物的平均值）
  - 引导用户导入历史记录
  - 前期以记录为主，分析逐步增强

### 4. 隐私合规
- **问题**：宠物数据也属于个人信息
- **缓解**：
  - 本地优先架构（数据不上云）
  - 端到端加密（可选云端同步）
  - 用户完全控制数据导出/删除
