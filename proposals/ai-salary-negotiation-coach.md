# AI 薪资谈判教练

> **一句话卖点**：跳槽者拿到 offer 后拍照上传，AI 3秒分析是否被压价，生成3套话术方案，帮助用户多拿30%薪资还留好印象

## 概述

基于 Issue #136 创建的详细设计文档。

薪资谈判是职场人最大的信息不对称场景：HR 有薪资数据、谈判经验、压价套路，而求职者往往是「裸奔」。本产品通过 AI 技术打破这种不对称，为用户提供数据支撑的话术策略和模拟训练。

**核心价值**：
- 数据支撑：市场薪资对比，知道「自己值多少钱」
- 话术生成：3套策略（保守/中等/激进），逐字稿直接用
- 模拟训练：AI扮演HR，提前练习应对各种压价套路

## 竞品分析

| 产品 | 功能 | 不足 | 我们的优势 |
|------|------|------|-----------|
| Boss直聘「薪资查询」 | 基础薪资区间展示 | 无个性化分析，无话术指导 | 提供完整谈判方案，不是只看数据 |
| OfferShow | 薪资爆料社区 | 数据质量参差不齐，无行动指导 | 结构化分析 + 可执行话术 |
| 脉脉「薪资对比」 | 匿名薪资分享 | 无谈判策略，偏社区属性 | 专注于「如何谈」，而非「别人拿多少」 |
| 付费职业咨询 | 一对一指导 | 价格高（¥500-2000），响应慢 | ¥19.9 即可获得专业级话术方案 |

**差异化定位**：
- 不是「薪资查询工具」→ 是「谈判行动教练」
- 不是「社区爆料」→ 是「结构化分析 + 可执行方案」
- 不是「人工咨询」→ 是「AI 秒级响应 + 个性化话术」

## 功能设计

### 核心功能

1. **市场薪资对标**
   - 基于职位/公司/城市，给出薪资区间和市场价分析
   - 数据来源：看准网、职友集、Boss直聘薪资报告
   - 置信度展示：数据量、更新时间、样本分布

2. **Offer 分析报告**
   - OCR 识别：提取 offer 中的薪资、福利、期权信息
   - **用户校对机制**：OCR 识别后展示提取结果，用户可手动修正（解决非标准格式准确性问题）
   - 对比分析：与市场薪资对比，计算「压价比例」
   - 风险评估：识别条款中的潜在陷阱（竞业协议、试用期薪资等）
   - 30秒快速输出：市场价对比 + 压价比例 + 推荐策略

3. **话术方案生成**
   - 三套策略：保守/中等/激进，每套都有逻辑支撑
   - 个性化调整：根据用户性格测试调整话术风格
   - 逐字稿：可直接复制使用，避免「有想法说不出」
   - HR 套路库：常见压价话术及对应破解方案

4. **谈判模拟训练**
   - AI 扮演 HR：模拟各种压价套路（「预算有限」「你资历不够」）
   - 实时对话：用户练习应对，AI 给出即时反馈
   - 评分系统：评估谈判技巧和进步曲线

5. **实时教练（通话模式）** - MVP 后期
   - 谈薪时实时提示应对策略
   - 骨传导耳机隐蔽使用

### 用户流程

```
上传Offer → 30秒分析 → 查看报告 → 选择策略 → 生成话术 → 模拟练习 → 实战谈判
```

## 技术方案

### 数据层

```
┌─────────────────────────────────────────┐
│            薪资数据源（合规路径）         │
├─────────────────────────────────────────┤
│ 优先级1：用户匿名贡献数据（飞轮效应）      │
│ 优先级2：公开政府统计数据                 │
│ 优先级3：第三方数据合作（官方API）         │
│ 待探索：招聘平台合作接口                  │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│         PostgreSQL + Redis              │
│ • 薪资数据表（职位/城市/公司/薪资区间）    │
│ • 用户脱敏数据                           │
│ • 话术模板库                             │
│ • HR套路库                               │
└─────────────────────────────────────────┘
```

**数据合规策略**：
- **MVP 阶段**：仅使用用户自愿贡献的匿名薪资数据 + 公开政府统计
- **增长阶段**：寻求与招聘平台、猎头公司的官方数据合作
- **数据质量**：明确每条数据的来源、样本量、更新时间、置信度
- **风险规避**：暂不爬取第三方网站数据，避免法律风险

**冷启动数据方案**（MVP 阶段关键）：
- **Fallback 策略**：用户贡献数据不足时，使用公开政府统计数据（国家统计局薪资报告、人社局行业薪资指导价）作为基准
- **数据透明**：明确标注「基于公开统计」或「基于 N 位用户贡献」，让用户了解数据来源和置信度
- **Beta 标签**：市场对标功能标记为「Beta 功能，数据持续积累中」，管理用户预期
- **冷启动加速**：首批 100 位用户免费使用高级功能，换取匿名薪资贡献

**技术选型理由**：
- **Node.js + TypeScript**：团队熟悉、生态丰富、与前端 Next.js 技术栈统一
- **GLM-4/Claude API**：国产 LLM 性价比高，Claude 在复杂推理场景表现更好
- **阿里云 OCR**：国内访问稳定，准确率高，价格透明

## 实现步骤

### Phase 1：MVP（3周）

- [ ] 薪资数据采集与清洗（含冷启动数据 fallback）
- [ ] Offer 文本解析（手动输入 + 简单 OCR）
- [ ] 薪资对比分析
- [ ] 3 套话术生成（基于模板 + LLM）
- [ ] Web 端基础界面
- [ ] **话术质量验证**：邀请 5-10 位 HR 专业人士和近期跳槽者内测，收集反馈优化话术模板

**MVP 验证指标**：
- 用户完成分析后的话术使用率
- 用户反馈「有帮助」比例 > 70%
- 7 日留存率 > 30%（用户会回来查看报告或使用其他功能）
- NPS（净推荐值）> 20（初期目标）
- 完成率 > 60%（从上传 Offer 到查看完整报告的转化）

### Phase 2：增强版（3周）

- [ ] OCR 自动识别（支持图片上传）
- [ ] 条款风险分析
- [ ] 用户性格测试 + 话术风格适配
- [ ] HR 套路库 + 破解话术

### Phase 3：模拟训练（3周）

- [ ] AI 谈判模拟器
- [ ] 实时对话反馈
- [ ] 谈判技巧评分

### Phase 4：实时教练（6周）- ⚠️ 高风险，独立验证后再投入

- [ ] 通话模式（技术难度高）
- [ ] 语音识别 + 实时建议
- [ ] 骨传导耳机适配

**风险评估**：
- **技术风险**：实时语音识别准确率受网络、方言影响大；延迟可能影响体验
- **隐私风险**：实时监听通话涉及敏感隐私，需明确法律合规边界
- **用户接受度**：多数用户可能不愿在谈薪时佩戴设备
- **验证策略**：先通过 Phase 3 模拟训练验证用户需求，收集反馈后再决定是否投入 Phase 4
- **替代方案**：提供「快速参考卡片」功能，用户可提前准备好关键话术要点

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 薪资数据不准确 | 用户按建议谈判失败，失去信任 | 多数据源交叉验证；明确数据来源和置信度；提供「数据更新时间」 |
| 话术过于激进 | offer 被撤回 | 提供3套策略供用户选择；风险提示；用户反馈迭代 |
| 隐私合规 | offer 包含敏感信息 | **数据脱敏方案**：上传时自动识别并遮蔽姓名、身份证号、电话；**存储周期**：原始图片 24 小时后删除，仅保留脱敏后的结构化数据；**用户授权**：明确告知数据用途，提供「仅本地处理」选项；**合规认证**：符合个人信息保护法要求 |
| 用户不回来反馈 | 无法建立数据飞轮 | 激励机制：「成功后分享经验，免费使用高级功能」 |
| 竞品进入 | Boss直聘等平台可能复制 | 深耕话术质量；建立用户社区；快速迭代 |
