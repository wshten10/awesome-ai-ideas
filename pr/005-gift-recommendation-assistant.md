# AI 礼物推荐助手 - 让每一份礼物都送到心坎里

> 基于关系画像和实时热点的智能礼物推荐系统，告别送礼焦虑。

## 功能设计

### 核心功能
1. **多维度画像系统**
   - 关系类型标签（家人/朋友/同事/恋人）
   - 喜好标签库（自动提取 + 手动补充）
   - 送礼历史记录（避免重复）
   - 预算偏好设置

2. **智能推荐引擎**
   - 场合匹配：生日/节日/纪念日/感谢/道歉
   - 预算筛选：智能分配，推荐性价比最优
   - 热点融合：结合当下流行趋势
   - 个性化排序：基于历史反馈优化

3. **多模态输入**
   - 拍照识别：书架、摆件、装饰风格 → 推断兴趣
   - 语音描述：自然语言输入对方特点
   - 社交媒体分析（可选授权）：
     - 授权页面：明确展示「将访问哪些数据」「如何使用」「保留多久」
     - 数据范围：仅提取礼物偏好相关信息，不获取私人通讯
     - 保留期限：默认 30 天，可手动删除
     - 一键撤销：设置页面提供「撤销社交授权」按钮，立即删除相关数据

4. **礼物故事分享**
   - 生成「为什么选这个礼物」的暖心文案
   - 社交分享模板，促进病毒传播

## 技术方案

### 数据层
- **用户画像存储**：PostgreSQL + pgvector（向量搜索）
- **礼物数据库**：自建 + 电商 API 联盟（淘宝/京东/亚马逊）
- **热点数据来源**：
  - 优先使用官方 API（如有）或与第三方数据服务商合作
  - 备选方案：自建礼物趋势库，通过用户行为数据推断热点
  - ⚠️ 合规说明：不使用违反平台服务条款的爬虫方式

### 逻辑层
- **推荐引擎**：协同过滤 + LLM 理解混合
- **多模态处理**：
  - 图片：CLIP/ViT 提取特征
  - 语音：Whisper ASR + NLU
- **实时更新**：Redis 缓存热点数据

### 展示层
- **移动端优先**：React Native / Flutter
- **Web 管理后台**：Next.js
- **分享页面**：静态生成，SEO 友好

## 实现步骤

### Phase 1: MVP (3-4周)
1. 基础画像系统：问卷式快速画像
2. 手动输入喜好 + 预算
3. 接入单一电商 API
   - 优先验证淘宝联盟 API 接入可行性（需网站备案、流量证明）
   - 备选方案：若审核不通过，先使用自建礼物数据库（手动录入热门商品）
4. 基础推荐算法（规则 + 简单协同过滤）

### Phase 2: 多模态 (3周)
1. 拍照识别兴趣（CLIP 模型）
2. 语音输入处理
3. 优化推荐算法（引入 LLM 理解）

### Phase 3: 社交增长 (2周)
1. 礼物故事生成
2. 分享功能 + 裂变激励
3. 节日营销自动化

### Phase 4: 商业化 (持续)
1. 订阅功能
2. 多电商平台对接
3. B2B 企业礼品定制

## 资源需求

### API & 服务
- **LLM API**：GLM-4 / GPT-4o-mini（理解用户需求）
- **多模态模型**：CLIP（图片理解）
- **语音识别**：Whisper API
- **电商联盟 API**：淘宝联盟、京东联盟、Amazon PA-API

### 基础设施
- **数据库**：PostgreSQL (Supabase/Railway)
- **向量存储**：pgvector
- **缓存**：Redis
- **CDN**：图片加速

### 预估成本（月）
- LLM API: $50-100
- 多模态处理: $30-50
- 基础设施: $20-40
- **总计: $100-200/月**

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 电商 API 政策变化 | 自建礼物数据库作为备选 |
| 隐私数据敏感 | 本地化存储选项，明确隐私政策 |
| 冷启动问题 | 问卷画像 + 热门礼物兜底 |
| 付费意愿待验证 | 先免费验证需求，再推订阅 |

## 成功指标

- **MVP 目标**：1000 用户，礼物推荐采纳率 > 60%
  - 「采纳」定义：用户点击推荐礼物的购买链接（最宽松）/ 完成实际购买（中等）/ 主动标记满意（最严格）
  - 追踪方式：MVP 阶段先追踪「点击购买链接」行为，Phase 2 引入用户反馈机制
- **Phase 2 目标**：日活 5000，分享率 > 15%
- **商业化目标**：3 个月内付费转化 > 5%
