# AI 破冰对话教练

> **一句话卖点**：社恐职场人在电梯遇到领导时，用 AI 实时推送 3 个破冰话题，从低头假装没看到到 30 秒聊出人脉机会

## 概述

基于 Issue #129 创建的详细设计文档。

职场社交焦虑是普遍问题：70% 的职场人有「社交焦虑」，「不知道说什么」是排名第一的社交障碍。本产品通过场景感知 + 关系分析 + 实时话题推荐，帮助社恐职场人从「想聊但不知道说什么」到「自信建立人脉」。

## 功能设计

### 核心功能

1. **场景感知**
   - 位置感知：GPS + 办公室地图（需要企业授权）
   - 日程感知：读取日历，知道今天有哪些会议/活动
   - 场景分类：电梯（30秒）、茶水间（5分钟）、会议室、活动

2. **关系分析**
   - 公开信息：从企业通讯录获取组织架构
   - 共同点挖掘：用 LLM 分析双方背景，找出交集（项目、兴趣、背景）
   - 兴趣推断：从公开动态分析兴趣爱好

3. **话题推荐**
   - 实时推送 3 个破冰话题
   - 话题分级：
     - Level 1：简单问候（「最近忙吗？」）
     - Level 2：找共同点（「听说你在做 XX 项目」）
     - Level 3：建立连接（「有机会一起喝咖啡聊聊」）
   - 后续跟进点：聊完这次，下次还能聊什么

4. **练习模式**
   - 模拟真实场景（电梯、茶水间、会议室）
   - AI 扮演不同的对象（领导、同事、跨部门）
   - 演练对话技巧，提升自信

5. **复盘功能**
   - 记录错失的机会，AI 分析原因
   - 「下次遇见提示」：标记想聊的人，下次遇到前推送话题
   - 社交能力成长曲线

## 技术方案

### 数据层

```
用户数据：
- 基本信息（姓名、部门、职位）
- 日程数据（从日历读取）
- 社交历史（聊天记录、复盘笔记）

关系数据：
- 组织架构（企业通讯录）
- 项目信息（企业内网/项目文档）
- 公开信息（LinkedIn/脉脉）

场景数据：
- 位置数据（GPS）
- 办公室地图（企业授权）
- 场景模板（电梯、茶水间、会议室）
```

### 逻辑层

```
1. 场景识别模块
   - 位置感知 → 场景分类
   - 日程感知 → 预测可能遇到的人
   - 时间紧迫度 → 话题长度适配

2. 关系分析模块
   - 组织架构 → 关系远近
   - 项目信息 → 共同话题
   - 公开信息 → 兴趣推断

3. 话题生成模块
   - LLM 生成 3 个话题（基于场景 + 关系 + 共同点）
   - 话题分级（Level 1/2/3）
   - 后续跟进点

4. 练习模式模块
   - AI 扮演对象（领导/同事/跨部门）
   - 对话模拟 + 反馈
```

### 展示层

```
移动端（主场景）：
- 首页：当前场景 + 推荐话题
- 话题详情：话题 + 背景信息 + 跟进点
- 练习模式：场景选择 + 对话模拟
- 复盘：历史记录 + 分析

Web 端（管理）：
- 社交网络分析：人脉结构可视化
- 进步追踪：社交能力成长曲线
- 设置：企业授权 + 数据管理
```

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| **隐私敏感**：读取企业通讯录需要授权 | 个人版用「手动输入」，企业版用「组织授权」；数据本地优先，端到端加密 |
| **依赖性问题**：用户过度依赖 AI 话题 | 设计「渐进式训练」，逐步培养用户自己的社交能力；提供「话题创作技巧」课程 |
| **话题质量**：话题太套路，用户觉得「假」 | 用真实案例训练，让话题听起来自然；用户反馈机制，持续优化话题质量 |
| **冷启动问题**：没有企业数据时，如何分析共同点？ | 先用公开数据（LinkedIn/脉脉）+ 手动输入；提供「话题模板库」作为备选 |
| **场景识别准确性**：GPS 信号弱，场景误判 | 结合多种信号（GPS + Wi-Fi + 日程）；用户可手动修正场景 |
| **企业授权门槛**：企业可能不愿开放通讯录 | 提供「个人版」无需授权；强调数据安全和隐私保护；企业版提供试用 |
