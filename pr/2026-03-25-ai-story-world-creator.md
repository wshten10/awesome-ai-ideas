# AI 故事世界共创师

> **一句话卖点**：下班后躺在沙发上的职场人，用 AI 10 秒进入角色扮演世界，从被动刷 2 小时视频看完就忘，到主动创造属于自己的故事世界

## 概述

基于 Issue #149 创建的详细设计文档。

现代人的娱乐方式陷入「被动消费」陷阱：刷 2 小时短视频，笑完就忘；想玩桌游 RPG 找不到队友；想体验剧情游戏受限于固定脚本。结果是：娱乐时间过去了，但什么都没留下，反而更空虚。

本产品通过自然语言角色扮演 + 动态世界生成 + 多角色互动，让用户从「消费故事」变成「创造故事」，每次冒险都是独一无二的体验。

## 功能设计

### 核心功能

1. **自然语言角色扮演**
   - 直接对话，无需学习规则
   - AI 理解用户意图并推进剧情
   - 支持任意角色设定（末世机械师、武侠剑客、星际船长...）

2. **动态世界生成**
   - 用户说「我想当海盗」→ 10 秒进入海盗船场景
   - AI 根据用户选择实时调整世界观和剧情走向
   - 每次选择都有蝴蝶效应，故事永不重复

3. **多角色互动**
   - AI 扮演多个 NPC，每个都有独特性格和背景
   - NPC 之间有互动，世界有生命力
   - 支持「剧情导演」模式，协调 NPC 行为避免矛盾

4. **故事保存分享**
   - 完成的冒险保存为「故事书」
   - 支持导出 Markdown/JSON 格式
   - 朋友可以续写你的故事

5. **多种题材**
   - 内置题材：末世生存、武侠江湖、科幻星际、悬疑推理、恋爱养成
   - 支持自定义世界观

6. **「随时暂停」机制**
   - 现实中有事可以随时暂停
   - 回来时「前情提要」快速回忆

### 差异化功能

| 现有方案 | 本产品 |
|---------|--------|
| 刷视频 2 小时，看完就忘 | 创作自己的故事，每次都不同 |
| 想玩桌游找不到人 | AI 24 小时在线，随时开始 |
| 剧情游戏固定分支 | AI 实时生成，无限可能 |
| 只能消费不能创造 | 边玩边创，故事属于你 |

   - 竞品：长对话容易遗忘设定

### 单位经济分析

| 指标 | 免费用户 | 订阅用户 |
|------|----------|----------|
| LLM 成本 | ¥2.0/月 | ¥5.0/月 |
| 分摊固定成本 | ¥0.23/月 | ¥0.23/月 |
| **总成本** | **¥2.23/月** | **¥5.23/月** |
| 收入 | ¥0 | ¥19.9/月 |
| **毛利** | **-¥2.23** | **¥14.67** |

**盈亏平衡点**：每 1 个付费用户需补贴 6.6 个免费用户（14.67 / 2.23 ≈ 6.6）

**LTV 计算**：
- 订阅用户平均留存：3 个月
- LTV = ¥19.9 × 3 = ¥59.7
- LTV/CAC = 59.7 / 25 = **2.39**（健康区间 2-4）

**结论**：¥19.9/月 定价合理，但需控制免费用户成本（限制轮次 + 本地模型）

## 内容安全方案

### 三层防护体系

**第 1 层：输入过滤（用户发送前）**
- **技术方案**：阿里云/腾讯云内容安全 API
- **检测内容**：
  - 敏感词（政治、暴力、色情）
  - 恶意 Prompt（越狱、角色扮演绕过）
- **处理方式**：拦截 + 提示「请修改内容」
- **成本**：¥0.002/次，月均 ¥500（1 万用户 × 25 次）

**第 2 层：输出过滤（AI 生成后）**
- **技术方案**：同上 API + 自研规则引擎
- **检测内容**：
  - AI 生成的敏感内容
  - 剧情崩坏（逻辑矛盾、价值观问题）
- **处理方式**：重新生成 + 记录日志
- **优化**：本地小模型预过滤，降低 API 调用

**第 3 层：人工审核（事后抽查）**
- **审核对象**：
  - 用户举报的故事
  - 高频互动故事（>50 轮对话）
  - 随机抽样（5% 新故事）
- **审核标准**：
  - 不涉及政治敏感
  - 不宣扬暴力/歧视
  - 符合社会主义核心价值观
- **处理流程**：
  - 违规故事：下架 + 警告用户
  - 严重违规：封禁账号
- **人力成本**：1 名兼职审核，¥3000/月

### 合规策略

- **备案**：ICP 备案 + 网络文化经营许可证
- **未成年人保护**：实名认证 + 时间限制
- **数据安全**：用户数据加密存储，7 天后删除对话记录

## 多 Agent 协调机制

### 架构设计

```
┌─────────────┐
│  剧情导演   │ ← 协调中心
└──────┬──────┘
       │
   ┌───┴───┬───────┬───────┐
   │       │       │       |
┌──▼──┐ ┌──▼──┐ ┌──▼──┐ ┌──▼──┐
│ NPC1│ │ NPC2│ │ NPC3│ │ 场景│
└─────┘ └─────┘ └─────┘ └─────┘
```

### 剧情导演职责

1. **一致性维护**
   - 记录关键事件（用户选择、NPC 行为、世界观变化）
   - 检查新内容是否与历史冲突
   - 冲突时：提示 NPC 修改或重新生成

2. **优先级规则**
   - **用户意图 > NPC 个性**：用户说「我想和医生合作」，医生不能无理由拒绝
   - **剧情推进 > NPC 自由**：NPC 行为需服务主线，不能跑题
   - **安全合规 > 一切**：任何内容不得违反安全规则

3. **冲突解决**
   - **NPC 间冲突**：剧情导演协调，让一方妥协或创造新剧情
   - **用户 vs NPC 冲突**：尊重用户选择，NPC 适应
   - **逻辑矛盾**：回溯最近 3 轮对话，找出矛盾点并修正

### 协调示例

**场景**：用户选择「攻击商人」，NPC 医生之前说「商人是好人」

- 剧情导演检测到冲突
- 协调方案：
  1. 医生改口「其实商人也有阴暗面」
  2. 或医生表示「我不认同你的做法」→ 增加角色深度
  3. 优先选择方案 2（更真实）
