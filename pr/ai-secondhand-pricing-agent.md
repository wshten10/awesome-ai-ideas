# AI 二手物品定价师

> **一句话卖点**：闲鱼卖家拍张照片，AI 3秒给出精准估价 + 生成吸睛描述，从纠结3天到10分钟上架卖出

## 概述

基于 Issue #162 创建。解决闲鱼/转转卖家的核心痛点——不知道该定多少钱，定高了卖不掉，定低了亏大了。

## 功能设计

### 核心功能
- **拍照估价**：拍物品照片 → AI 分析品相（全新/99新/95新/战损版）→ 3秒给出价格区间
- **智能定价**：基于近期成交数据（7/30/90天）给出「急出价」vs「等待价」
- **描述生成**：自动生成标题 + 卖点 + 关键词，提升曝光率
- **价格追踪**：监控同类商品价格变化，动态调价建议

### 辅助功能
- **历史成交参考**：展示 10 条同类商品成交记录
- **平台对比**：闲鱼/转转/小红书价格差异分析
- **上架提醒**：最佳上架时间推荐（周末/晚间高峰）

## 技术方案

### 数据层
- **价格数据库**：PostgreSQL 存储历史成交数据
- **向量数据库**：Milvus/Pinecone 存储商品特征向量（相似商品检索）
- **数据来源**（合规优先）：
  1. **官方 API 合作**：优先与闲鱼/转转官方建立数据合作关系
  2. **用户授权数据**：用户主动上传成交截图，获得使用额度
  3. **公开搜索数据**：利用平台公开搜索功能（非爬取后台）
  4. **合作伙伴数据**：与爱回收等回收平台合作获取价格参考
- **缓存层**：Redis 缓存热门商品价格

### 逻辑层
- **多模态识别**：GPT-4V/GLM-4V 识别物品类别、品牌、品相
- **价格预测模型**：
  - 协同过滤（相似商品定价）
  - 时间序列分析（价格趋势）
  - 规则引擎（品牌溢价、季节因素）
- **描述生成**：LLM 生成吸睛标题 + 卖点描述

### 展示层
- **移动端优先**：微信小程序 + App
- **核心交互**：拍照 → 3秒估价 → 一键上架
- **可视化**：价格走势图、成交概率预测

## 实现步骤

### Phase 1：MVP（3-4 周）
1. **单品类验证**：聚焦手机/数码品类（市场大、标准化程度高）
2. **基础估价**：拍照 → 品牌识别 → 基础估价
3. **描述生成**：自动生成标题 + 3 个卖点
4. **冷启动数据**：
   - **众包数据**：用户上传成交截图，获得免费使用额度
   - **公开数据**：利用闲鱼公开搜索结果（不爬取后台）
   - **合作数据**：与二手回收平台（爱回收）合作获取价格参考
   - **种子用户贡献**：前 100 名用户提供 10 条成交数据换 VIP

### Phase 2：多品类扩展（4周）
1. 扩展至潮鞋/包包/图书/家电
2. 多平台数据整合（闲鱼+转转+小红书）
3. 智能定价策略（急出价 vs 等待价）
4. 价格追踪 + 动态调价建议

### Phase 3：商业化（4周）
1. 批量估价功能（专业卖家）
2. 店铺版（多账号管理）
3. 与回收平台合作（B2B2C）
4. 数据服务（向品牌方提供二手市场行情）

## 成本分析

### 单用户成本（估算）
- **API 调用成本**：
  - 多模态识别（1 次）：¥0.02
  - LLM 生成（1 次）：¥0.01
  - 单次估价成本：¥0.03
- **存储成本**：¥0.001/用户/月（商品特征向量 + 历史记录）
- **计算成本**：¥0.005/用户/月（价格预测、相似商品检索）
- **总成本**：¥0.036/次估价 或 ¥0.036 + ¥0.006/月（活跃用户）

### 盈亏平衡分析
- **Freemium 模式**：
  - 免费用户：成本 ¥0.036/次，无收入 → 需限制免费次数（如 3 次/天）
  - 付费用户：¥9.9/次，利润率 99.6%（¥9.864）
- **订阅模式**：
  - ¥29.9/月，假设 10 次估价/月，成本 ¥0.36，利润率 98.8%（¥29.54）
- **盈亏平衡点**：
  - 每个付费用户覆盖约 830 个免费用户的成本（¥29.9 / ¥0.036）

### 规模化成本优势
- **用户量 1 万**：月成本 ~¥600（假设人均 5 次估价）
- **用户量 10 万**：月成本 ~¥6000，可考虑自建模型降低 API 成本

## 风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| **法律合规**（重要） | 优先寻求官方 API 合作；使用用户授权数据；避免爬取平台后台数据 |
| 平台反爬 | 优先官方 API，建立数据合作；用户众包数据作为补充 |
| 数据准确性 | 价格波动大时降低置信度；展示「同类商品成交价」作为依据 |
| 用户信任 | 提供「估价依据」（10 条成交记录）；建立反馈闭环 |
| 竞品跟进 | 建立数据壁垒（成交价格数据库）；品牌/品相识别模型 |
| 品类扩展难度 | 先聚焦高频品类（手机/潮鞋）；逐步扩展 |
| 数据冷启动 | 众包模式 + 合作数据 + 种子用户贡献（见 MVP 阶段方案） |

## 竞品分析

| 竞品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------------|
| **闲鱼官方估价**（手机回收） | 官方数据、用户信任 | 仅限手机回收；不面向 C2C 卖家；价格偏低（回收价） | 面向 C2C 卖家；估价更高；描述生成一体化 |
| 爱回收 | 专业 | 只收不卖 | 估价 + 上架一体化 |
| 通用 ChatGPT | 免费 | 不懂市场行情 | 基于真实成交数据 |
| 小红书搜索 | 真实案例 | 需要手动筛选 | AI 自动分析 |

### 与闲鱼官方估价的差异化
闲鱼官方估价主要面向**手机回收场景**（B2C），价格是回收商收购价，通常低于市场价 20-30%。我们的差异化：
1. **面向 C2C 卖家**：估价是卖给其他用户的价格，比回收价高 20-40%
2. **描述生成一体化**：不仅估价，还生成吸睛标题 + 卖点描述
3. **多品类覆盖**：不止手机，还覆盖潮鞋、包包、图书、家电等
4. **价格追踪**：监控价格变化，动态调价建议（闲鱼官方无此功能）

## 成功指标

### 北极星指标
- **GMV 助力**：帮助用户多卖的总金额（目标：1000 万/月）

### 关键指标
- **估价准确率**：估价与实际成交价偏差 < 15%（目标：80%）
- **用户留存**：7 日留存 > 40%
- **转化率**：估价 → 实际上架 > 60%
- **NPS**：净推荐值 > 50
