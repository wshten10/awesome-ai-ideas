# AI 购物决策助手

> **一句话卖点**：精打细算的网购达人在下单前，用 AI 助手 3 秒识别刷评水军，发现真实低价，每笔订单省下 30%

## 概述

基于 Issue #32 创建的详细设计文档。

网购时面对海量商品和真假难辨的评价，消费者往往陷入决策瘫痪：同类商品太多不知道哪个真正适合自己、评价区水军刷评难辨、历史价格波动大、不同平台比价耗时费力。本产品通过 AI 评价分析 + 多平台比价 + 价格趋势追踪，帮助消费者从「选择困难症」到「3 秒下单省钱省心」。

## 功能设计

### 核心功能

1. **需求分析**
   - 对话式理解真实需求（预算、使用场景、偏好）
   - 智能推荐合适品类和参数
   - 避免买错/买贵/买多了

2. **多平台比价**
   - 实时对比主流电商平台（淘宝、京东、拼多多、抖音）
   - 显示各平台价格 + 优惠 + 运费
   - 一键跳转最低价平台

3. **评价可信度分析**
   - 识别刷评、水军、虚假好评
   - 筛选真实用户反馈
   - 评价质量评分（1-5 星）
   - 可信度标签：「真实买家」「疑似水军」「未验证」

4. **价格趋势追踪**
   - 历史价格曲线（30 天/90 天/365 天）
   - 判断是否最佳入手时机
   - 降价提醒：设置目标价，自动通知

5. **性价比评分**
   - 综合价格、质量、口碑的智能打分
   - 同类商品横向对比
   - 「省钱指数」：显示比平均价省了多少

6. **省钱报告**
   - 月度省钱统计
   - 分享炫耀：「我省了 ¥XXX」
   - 购物习惯分析

## 技术方案

### 数据层

```
商品数据：
- 基础信息（名称、价格、参数）
- 价格历史（各平台价格变化）
- 库存状态

评价数据：
- 原始评价（文本、图片、视频）
- 评价者信息（购买验证、历史评价）
- 可信度标签

用户数据：
- 购物偏好
- 价格提醒设置
- 省钱记录
```

### 逻辑层

```
1. 需求分析模块
   - 对话式需求收集
   - 智能推荐（品类 + 参数）
   - 预算优化

2. 比价引擎
   - 多平台实时比价
   - 优惠计算（满减 + 券 + 平台补贴）
   - 最低价推荐

3. 评价分析模块
   - NLP 分析评价真实性
   - 语言模式识别（刷评特征）
   - 购买验证检查
   - 可信度评分

4. 价格预测模块
   - 历史价格分析
   - 促销周期预测（618、双11）
   - 最佳入手时机推荐
```

### 展示层

```
移动端（主场景）：
- 首页：搜索框 + 热门品类
- 商品详情：比价 + 评价分析 + 价格趋势
- 省钱报告：月度统计 + 分享

Web 端（管理）：
- 价格提醒管理
- 历史购物分析
- 设置：平台偏好 + 通知

浏览器插件：
- 一键比价（浏览商品页时自动触发）
- 评价可信度提示
- 降价提醒弹窗
```

## 实现步骤

### Phase 1（MVP - 2 周）

**目标**：验证核心价值「比价 + 基础评价筛选」

- [ ] 商品搜索 + 基础信息展示
- [ ] **单平台比价验证（京东）** — 先验证核心流程，再扩展多平台
- [ ] **评价分析范围明确**：
  - ✅ 购买验证检查（是否为已购买用户评价）
  - ✅ 评价时效筛选（优先展示近期评价）
  - ✅ 图片/视频评价优先（真实性更高）
  - ❌ 暂不做复杂刷评识别（Phase 2 实现）
- [ ] 基础 UI（移动端 H5，非原生）

**技术栈**：
- 前端：React H5（降低开发成本，快速验证）
- 后端：Node.js + Express
- 数据源：**优先官方 API**（京东开放平台），第三方 API 作为备选
- 数据库：PostgreSQL

| 月活用户 | 10,000 | 10,000 |

| 点击 → 购买转化率 | 5% | 10% |

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| **平台反爬政策**：电商平台可能封杀爬虫 | 优先接入官方 API（京东开放平台）；备选第三方数据源；建立数据备份机制 |
| **数据源依赖风险**：过度依赖第三方 API，议价能力弱 | **分阶段数据策略**：(1) MVP 用官方 API；(2) 建立自有爬虫能力；(3) 探索与平台官方合作；保持多源备份 |
| **数据获取稳定性**：价格数据可能不及时 | 多数据源交叉验证；实时性标注；用户可手动刷新 |
| **评价分析准确性**：刷评手段不断升级 | 持续优化算法；用户反馈机制；建立刷评特征库 |
| **联盟佣金依赖**：平台可能降低佣金比例 | 多元化变现（订阅 + B2B）；不过度依赖单一平台 |
| **竞争激烈**：已有慢慢买、什么值得买等竞品 | 差异化：真实评价过滤 + 智能推荐；更好的用户体验 |
| **用户信任**：需要用户相信推荐是客观的 | 透明化分析过程；不收商家推广费；用户可验证结果 |
| **法律合规**：数据爬取、评价分析的合规性 | 见下方「合规风险」章节 |

### 合规风险

| 合规领域 | 风险点 | 应对措施 |
|----------|--------|----------|
| **数据爬取** | 违反平台 ToS，可能被封禁或法律风险 | 优先使用官方 API；爬虫仅用于公开数据；遵守 robots.txt；咨询法务 |
| **用户隐私** | 收集购物偏好、浏览记录等敏感数据 | 最小化数据收集；明确隐私政策；用户可删除数据；本地化存储敏感信息 |
| **评价分析** | 分析用户评价可能涉及隐私 | 仅分析公开评价；不存储原始评价文本；输出聚合结果 |
| **联盟推广** | 需要明示商业关系 | 在跳转前提示「通过联盟链接跳转」；符合广告法要求 |
| **平台政策** | 各平台规则不同，可能随时变化 | 持续监控政策变化；建立合规预警机制；预留政策适配时间 |
