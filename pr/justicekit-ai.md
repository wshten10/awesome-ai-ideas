# JusticeKit AI - AI驱动的自助诉讼导航与文书生成助手

## 📋 项目背景与用户痛点

### 社会价值评估
- **解决实际问题**：降低法律门槛，促进司法公正
- **普惠价值**：为自诉当事人提供专业法律帮助
- **效率提升**：简化复杂的法律流程，从数周到数天
- **可及性改善**：让普通人获得原本难以企及的法律服务

### 核心痛点深度分析

#### 1. 法律信息壁垒（影响：极高）
- **300万+** pro se案件（无律师代理的诉讼）每年在美国法院系统中处理
- 普通人面对法律文书如同"天书"——合同条款、诉讼格式、证据要求让人望而却步
- 平均每个自诉案件因文书错误导致 **$3,500-8,500** 额外法律费用
- 23%的案件因文档填写错误需要重新处理

#### 2. 时间与经济成本（影响：高）
- 传统法律服务收费 $200-500/小时，普通案件总费用 $5,000-20,000
- 法律援助覆盖面仅约 20% 的合格申请人
- 诉讼流程平均耗时 6-18个月，期间当事人需反复往返法院
- 低收入人群完全无法负担基本的法律服务

#### 3. 流程黑箱（影响：高）
- 诉讼流程涉及数十个步骤，每个步骤都有严格的格式要求和截止日期
- 错过一个截止日期可能导致案件被驳回
- 不同法院的流程差异大，缺乏统一指引
- 证据规则复杂，普通人难以判断什么证据有效

#### 4. 语言与文化障碍（影响：中高）
- 移民群体、少数族裔面临双重语言障碍
- 法律术语与日常用语差距巨大
- 文化差异导致对法律体系的不信任和误解

### 技术实现细节

#### 1. 法律知识图谱构建
```python
class LegalKnowledgeGraph:
    """法律知识图谱核心引擎"""
    
    def __init__(self):
        self.graph_db = Neo4jConnection()  # 图数据库存储法律关系
        self.vector_store = QdrantClient()  # 向量检索相似案例
        self.rule_engine = RuleEngine()     # 法律规则推理引擎
    
    def query_legal_process(self, case_type, jurisdiction):
        """查询特定案件类型的完整法律流程"""
        process = self.graph_db.get_process_flow(case_type, jurisdiction)
        requirements = self.graph_db.get_requirements(process.id)
        deadlines = self.graph_db.get_deadlines(process.id)
        return LegalProcessGuide(process, requirements, deadlines)
    
    def find_similar_cases(self, case_description, top_k=5):
        """检索相似历史案例"""
        embedding = self.embed(case_description)
        similar = self.vector_store.search(embedding, top_k=top_k)
        return [Case(c.metadata) for c in similar]
```

#### 2. 智能文书生成引擎
```python
class DocumentGenerator:
    """AI驱动的法律文书生成"""
    
    def generate_pleading(self, user_input, case_context):
        """生成诉状类文书"""
        # Step 1: 信息提取与结构化
        entities = self.extract_legal_entities(user_input)
        claims = self.identify_legal_claims(entities, case_context)
        
        # Step 2: 法律依据匹配
        legal_bases = self.match_legal_provisions(claims)
        
        # Step 3: 文书结构化生成
        document = {
            'header': self.generate_header(case_context),
            'facts': self.generate_fact_statement(entities),
            'claims': self.generate_claim_section(claims, legal_bases),
            'evidence': self.generate_evidence_list(entities),
            'conclusion': self.generate_conclusion(claims),
            'signatures': self.generate_signature_block()
        }
        
        # Step 4: 合规性检查
        compliance = self.check_compliance(document)
        if compliance.has_issues:
            document = self.fix_compliance(document, compliance.issues)
        
        return document
```

**合规技术框架：**
```python
class ComplianceEngine:
    """三级法律合规保障"""
    
    def validate_output(self, document, case_type):
        # 第一层：AI自检
        issues = self.self_check(document)
        
        # 第二层：规则引擎验证
        rule_issues = self.rule_engine.validate(document, case_type)
        
        # 第三层：高风险文件标记
        if self.is_high_risk(document):
            document.status = 'REQUIRES_LAWYER_REVIEW'
            document.disclaimer = '本文件需经执业律师审核后方可使用'
        
        return ComplianceReport(issues, rule_issues)
```
