## 概述

随着AI代理工具的爆发式增长（OpenAI、Claude、Gemini、Cursor等），每个工具都拥有独立的记忆系统，导致用户认知碎片化。本项目旨在创建一个统一的记忆编排器，让用户在不同AI代理间实现知识连贯性，打造「一个AI大脑，多个AI身体」的协作生态。

## 🔍 用户痛点

- **记忆孤岛**：Claude不记得ChatGPT的对话，Cursor不记得IDE中的代码修改
- **重复劳动**：每次都要重新解释项目背景、个人偏好、上下文信息
- **认知负担**：需要手动同步各个AI工具之间的理解
- **信任赤字**：不同AI对同一用户有不同理解，导致建议不一致

## 🎯 核心功能

### 1. 统一记忆层 (Unified Memory Layer)
- **持久化记忆存储**：SQLite + 向量数据库，支持跨工具记忆同步
- **记忆版本控制**：Git-style记忆版本管理，可回溯和对比
- **智能记忆压缩**：自动识别冗余信息，保留核心价值记忆
- **隐私保护**：本地优先部署，支持端到端加密

### 2. 多协议适配器 (Multi-Protocol Adapters)
- **OpenAI Compatible API**：兼容ChatGPT/Claude API的记忆扩展
- **MCP Server集成**：与Model Context Protocol深度集成
- **IDE插件**：VSCode/IntelliJ插件，实时同步编码上下文
- **浏览器扩展**：网页内容自动记忆和分析

### 3. 智能记忆路由 (Smart Memory Routing)
- **情境感知**：根据使用场景自动激活相关记忆
- **优先级管理**：重要记忆实时同步，次要记忆压缩存储
- **冲突解决**：当多个AI工具产生冲突记忆时的智能仲裁
- **学习模式**：识别用户偏好，自动调整记忆权重

### 4. 记忆可视化仪表板
- **记忆图谱**：可视化的知识关系网络
- **使用热力图**：记忆使用频率和重要性分析
- **协作视图**：多AI工具间的知识流转可视化
- **健康监控**：记忆系统性能和健康度指标

## 🏗️ 技术方案

### 架构设计
用户界面层 -> 协议适配层 -> 核心记忆层

### 技术栈选择
- **后端**：Rust (性能、内存安全) + Python (AI集成)
- **数据库**：SQLite (本地存储) + Qdrant/HNSW (向量搜索)
- **前端**：React + TypeScript + D3.js (可视化)
- **通信**：gRPC + WebSocket (实时同步)
- **部署**：Docker + Docker Compose (一键部署)

### 数据模型
- 记忆项：内容、元数据、嵌入向量、版本信息
- 会话管理：用户ID、活跃工具、记忆状态、冲突解决
